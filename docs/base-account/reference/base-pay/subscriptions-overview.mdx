Определено в [Base Account SDK](https://github.com/base/account-sdk)

<Info>
Base Subscriptions позволяют принимать периодические платежи USDC. Пользователи предоставляют вашему приложению разрешение на периодическое списание средств со своего аккаунта, устраняя необходимость ручного одобрения каждого платежа. **Никаких комиссий для мерчантов или пользователей.**
</Info>

## Как работают подписки
<Steps>
<Step title="Пользователь создает подписку">
Пользователь одобряет разрешение на расходы для вашего приложения на списание определенной суммы периодически.
</Step>

<Step title="Приложение периодически списывает средства">
Ваш бэкенд списывает средства с подписки, когда наступает срок платежа, в пределах разрешенной суммы за период.
</Step>

<Step title="Автоматический сброс периода">
Лимит расходов автоматически сбрасывается в начале каждого нового периода.
</Step>

<Step title="Пользователь может отменить в любое время">
Пользователи сохраняют полный контроль и могут отозвать разрешение в любое время.
</Step>
</Steps>

## Определения типов

```typescript
// Параметры создания подписки
interface SubscriptionOptions {
  recurringCharge: string;
  subscriptionOwner: string;
  periodInDays?: number;
  testnet?: boolean;
  telemetry?: boolean;
}

// Результат подписки
interface SubscriptionResult {
  id: string;
  subscriptionOwner: Address;
  subscriptionPayer: Address;
  recurringCharge: string;
  periodInDays: number;
}

// Статус подписки
interface SubscriptionStatus {
  isSubscribed: boolean;
  recurringCharge: string;
  remainingChargeInPeriod?: string;
  currentPeriodStart?: Date;
  nextPeriodStart?: Date;
  periodInDays?: number;
}

// Подготовка списания
interface PrepareChargeOptions {
  id: string;
  amount: string | 'max-remaining-charge';
  testnet?: boolean;
}

type PrepareChargeResult = Array<{
  to: Address;
  data: Hex;
  value: '0x0';
}>;
```
