---
title: "eth_getLogs"
description: "Получить массив всех логов, соответствующих заданному объекту фильтра"
---

Определено в [Спецификации Ethereum JSON-RPC](https://ethereum.org/en/developers/docs/apis/json-rpc/)

<Info>
Возвращает массив всех логов, соответствующих заданному объекту фильтра.
</Info>

## Параметры

<ParamField body="filterObject" type="object" required>
Объект параметров фильтра.

<Expandable title="Свойства объекта фильтра">
<ParamField body="fromBlock" type="string">
Номер блока в виде целого числа, "latest" для последнего добытого блока или "pending", "earliest" для еще не добытых транзакций.
</ParamField>

<ParamField body="toBlock" type="string">
Номер блока в виде целого числа, "latest" для последнего добытого блока или "pending", "earliest" для еще не добытых транзакций.
</ParamField>

<ParamField body="address" type="string | array">
Адрес контракта или список адресов, из которых должны происходить логи.
</ParamField>

<ParamField body="topics" type="array">
Массив тем (topics) размером 32 байта каждая. Темы зависят от порядка.
</ParamField>

<ParamField body="blockhash" type="string">
Использование blockHash эквивалентно fromBlock = toBlock = номеру блока с хэшем blockHash.
</ParamField>
</Expandable>
</ParamField>

## Возвращает

<ResponseField name="result" type="array">
Массив объектов логов, соответствующих фильтру.

<Expandable title="Свойства объекта лога">
<ResponseField name="removed" type="boolean">
True - если лог был удалён из-за реорганизации цепи. False - если это действительный лог.
</ResponseField>

<ResponseField name="logIndex" type="string">
Позиция индекса лога в блоке (hex). null для ожидающего лога.
</ResponseField>

<ResponseField name="transactionIndex" type="string">
Позиция индекса транзакции, из которой был создан лог (hex). null для ожидающего лога.
</ResponseField>

<ResponseField name="transactionHash" type="string">
Хэш транзакции, из которой был создан лог (hex). null для ожидающего лога.
</ResponseField>

<ResponseField name="blockHash" type="string">
Хэш блока, в котором находился этот лог (hex). null для ожидающего лога.
</ResponseField>

<ResponseField name="blockNumber" type="string">
Номер блока, в котором находился лог (hex). null для ожидающего лога.
</ResponseField>

<ResponseField name="address" type="string">
Адрес, из которого произошёл лог.
</ResponseField>

<ResponseField name="data" type="string">
Неиндексированные параметры лога (hex).
</ResponseField>

<ResponseField name="topics" type="array">
Массив от 0 до 4 тем (topics) размером 32 байта каждая - индексированные аргументы лога.
</ResponseField>
</Expandable>
</ResponseField>

<RequestExample>
```json Basic Filter
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "eth_getLogs",
  "params": [{
    "fromBlock": "0x1",
    "toBlock": "latest",
    "address": "0xa0b86a33e6776e1e627e5c82df4c0cf77b8bb0c9"
  }]
}
```

```json With Topics
{
  "id": 1,
  "jsonrpc": "2.0",

      "0xddf252ad1be2c89b69c2b068fc378daa952ba7f163c4a11628f55a4df523b3ef"
    ]
  }]
}
```
</RequestExample>

## Обработка ошибок

| Код    | Сообщение                      | Описание    |
| ------ | ------------------------------ | ----------- |
| -32602 | Invalid filter parameters     | Объект фильтра содержит некорректные параметры |
| -32005 | Limit exceeded                 | Фильтр соответствует слишком большому количеству логов |
| 4100   | Requested method not supported | Метод не поддерживается кошельком |

<Warning>
Большие диапазоны фильтров могут превышать лимиты провайдера. Рекомендуется использовать меньшие диапазоны блоков для лучшей производительности.
</Warning>

<Info>
Темы (topics) зависят от порядка. Используйте `null` в качестве подстановочного знака для любой позиции темы.
</Info>
