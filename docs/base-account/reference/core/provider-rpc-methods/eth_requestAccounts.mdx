---
title: "eth_requestAccounts"
description: "Запросить доступ к аккаунтам и авторизацию у пользователя"
---

Определено в [EIP-1102](https://github.com/ethereum/EIPs/blob/master/EIPS/eip-1102.md)

<Info>
Запрашивает у пользователя предоставление адреса Ethereum для идентификации. Этот метод отображает пользователю запрос на авторизацию подключения. Возвращает промис, который разрешается массивом аккаунтов, авторизованных пользователем для приложения.
</Info>

## Параметры

Этот метод не принимает параметров.

## Возвращает

<ResponseField name="result" type="Array<string>">
Массив адресов Ethereum (шестнадцатеричные строки), которые пользователь авторизовал для приложения. Обычно массив содержит один адрес - текущий выбранный аккаунт в кошельке.
</ResponseField>

<RequestExample>
```json Request
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "eth_requestAccounts",
  "params": []
}
```
</RequestExample>

<ResponseExample>
```json Success Response
{
  "id": 1,
  "jsonrpc": "2.0",
  "result": ["0xabc123..."]
}
```
</ResponseExample>

## Обработка ошибок

| Код  | Сообщение                      | Описание                                                |
| ---- | ------------------------------ | ------------------------------------------------------- |
| 4001 | User rejected the request      | Пользователь отклонил запрос на подключение                  |
| 4100 | Requested method not supported | Провайдер не поддерживает метод `eth_requestAccounts` |
| 4200 | Wallet not connected           | Кошелек недоступен или не подключен             |

<Warning>
Всегда корректно обрабатывайте в приложении случай, когда пользователь отклоняет запрос на подключение (код ошибки 4001).
</Warning>
