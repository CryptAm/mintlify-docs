---
title: "eth_getCode"
description: "Получить байт-код по указанному адресу"
---

Определено в [Спецификации Ethereum JSON-RPC](https://ethereum.org/en/developers/docs/apis/json-rpc/)

<Info>
Возвращает байт-код по указанному адресу. Полезно для проверки, является ли адрес контрактом или внешним аккаунтом (EOA).
</Info>

## Параметры

<ParamField body="address" type="string" required>
Адрес, с которого требуется получить байт-код (20 байт).
</ParamField>

<ParamField body="blockParameter" type="string" required>
Номер блока в виде целого числа или строки "latest", "earliest" или "pending".
</ParamField>

## Возвращает

<ResponseField name="result" type="string">
Байт-код по указанному адресу в виде шестнадцатеричной строки (hex). Возвращает "0x" для внешних аккаунтов (EOA).
</ResponseField>

<RequestExample>
```json Contract Address
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "eth_getCode",
  "params": [
    "0xa0b86a33e6776e1e627e5c82df4c0cf77b8bb0c9",
    "latest"
  ]
}
```

```json EOA Address
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "eth_getCode",
  "params": [
    "0x407d73d8a49eeb85d32cf465507dd71d507100c1",
    "latest"
  ]
}
```
</RequestExample>

<ResponseExample>
```json Contract Response
{
  "id": 1,
  "jsonrpc": "2.0",
  "result": "0x608060405234801561001057600080fd5b50600436106100365760003560e01c..."
}
```

```json EOA Response
{
  "id": 1,
  "jsonrpc": "2.0",
  "result": "0x"
}
```
</ResponseExample>

## Обработка ошибок

| Код    | Сообщение                          | Описание    |
| ------ | ---------------------------------- | ----------- |
| -32602 | Invalid address or block parameter | Указан некорректный адрес или параметр блока |
| 4100   | Requested method not supported     | Метод не поддерживается кошельком |

<Info>
Если результат равен "0x", адрес является внешним аккаунтом (EOA). Если он содержит байт-код, это смарт-контракт.
</Info>
