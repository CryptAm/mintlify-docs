Экспериментальный RPC метод для получения суб-аккаунтов

<Info>
Получает суб-аккаунты, связанные с основным аккаунтом кошелька. Суб-аккаунты позволяют осуществлять иерархическое управление аккаунтами и делегирование прав.
</Info>

## Параметры

<ParamField body="account" type="string" required>
Адрес основного аккаунта кошелька.

<ParamField body="domain" type="string" required>
Полное доменное имя приложения, с которым связан суб-аккаунт.
</ParamField>

</ParamField>

## Возвращает

<ResponseField name="result" type="object" >
Sub account information including address and deployment details.

<Expandable title="Свойства результата">

<ResponseField name="subAccounts" type="array">
Массив информации о суб-аккаунтах.

<Expandable title="Свойства суб-аккаунта">

<ResponseField name="address" type="string">
Адрес созданного суб-аккаунта.
</ResponseField>

<ResponseField name="factory" type="string">
Адрес фабричного контракта.
</ResponseField>

<ResponseField name="factoryData" type="string">
Данные развертывания фабрики.
</ResponseField>

</Expandable>
</ResponseField>

</Expandable>
</ResponseField>

<RequestExample>
```json Request
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "wallet_getSubAccounts",
  "params": [{
    "account": "0x1234567890123456789012345678901234567890",
    "domain": "https://app.example.com"
  }]
}
```
</RequestExample>

<ResponseExample>
```json Success Response
{
  "id": 1,
  "jsonrpc": "2.0",
  "result": {
    "subAccounts": [
      {
        "address": "0x2345678901234567890123456789012345678901",
        "factory": "0x1234567890123456789012345678901234567890",
        "factoryData": "0x1234567890123456789012345678901234567890"
      }
    ]
  }
}
```
</ResponseExample>

## Обработка ошибок

| Код  | Сообщение                      | Описание    |
| ---- | ------------------------------ | ----------- |
| 4100 | Requested method not supported | Метод не поддерживается кошельком |
| -32602 | Invalid params                 | Некорректная конфигурация аккаунта |
