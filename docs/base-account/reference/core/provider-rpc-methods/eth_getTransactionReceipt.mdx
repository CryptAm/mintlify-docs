---
title: "eth_getTransactionReceipt"
description: "Получить квитанцию транзакции по хэшу транзакции"
---

Определено в [Спецификации Ethereum JSON-RPC](https://ethereum.org/en/developers/docs/apis/json-rpc/)

<Info>
Возвращает квитанцию транзакции по её хэшу. Обратите внимание, что квитанция недоступна для ожидающих транзакций.
</Info>

## Параметры

<ParamField body="transactionHash" type="string" required>
Хэш транзакции (32 байта).
</ParamField>

## Возвращает

<ResponseField name="result" type="object">
Объект квитанции транзакции или `null`, если квитанция не найдена.

<Expandable title="Свойства квитанции транзакции">
<ResponseField name="transactionHash" type="string">
Хэш транзакции.
</ResponseField>

<ResponseField name="transactionIndex" type="string">
Позиция индекса транзакции в блоке.
</ResponseField>

<ResponseField name="blockHash" type="string">
Хэш блока, в котором находилась эта транзакция.
</ResponseField>

<ResponseField name="blockNumber" type="string">
Номер блока, в котором находилась эта транзакция.
</ResponseField>

<ResponseField name="from" type="string">
Адрес отправителя.
</ResponseField>

<ResponseField name="to" type="string">
Адрес получателя. `null` для транзакции создания контракта.
</ResponseField>

<ResponseField name="cumulativeGasUsed" type="string">
Общее количество газа, использованного при выполнении этой транзакции в блоке.
</ResponseField>

<ResponseField name="gasUsed" type="string">
Количество газа, использованного только этой конкретной транзакцией.
</ResponseField>

<ResponseField name="contractAddress" type="string">
Адрес созданного контракта, если транзакция была созданием контракта, иначе `null`..
</ResponseField>

<ResponseField name="logs" type="array">
Массив объектов логов, сгенерированных этой транзакцией.
</ResponseField>

<ResponseField name="logsBloom" type="string">
Bloom-фильтр для логов транзакции.
</ResponseField>

<ResponseField name="status" type="string">
1 (успех) или 0 (неудача).
</ResponseField>
</Expandable>
</ResponseField>

## Обработка ошибок

| Код    | Сообщение                      | Описание    |
| ------ | ------------------------------ | ----------- |
| -32602 | Invalid transaction hash       | Указан некорректный хэш транзакции |
| 4100   | Requested method not supported | Метод не поддерживается кошельком |

<Info>
Квитанции транзакций доступны только для добытых транзакций. Для ожидающих транзакций возвращается `null`.
</Info>
