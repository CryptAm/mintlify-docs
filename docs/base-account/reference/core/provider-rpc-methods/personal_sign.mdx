Определено в [EIP-191](https://eips.ethereum.org/EIPS/eip-191)

<Info>
Подписывает данные с использованием определенного аккаунта. Этот метод вычисляет Ethereum-специфичную подпись по формуле: `sign(keccak256("\x19Ethereum Signed Message:\n" + len(message) + message)))`.
</Info>

## Параметры

<ParamField body="data" type="string" required>
Данные для подписи в шестнадцатеричном формате (hex).
</ParamField>

<ParamField body="address" type="string" required>
Адрес аккаунта, который должен подписать данные.
</ParamField>

## Возвращает

<ResponseField name="result" type="string">
Строка подписи в шестнадцатеричном формате (hex).
</ResponseField>

## Пример использования

<RequestExample>
```json Request
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "personal_sign",
  "params": [

  ]
```

## Обработка ошибок

<ResponseField name="code" type="number">
Код ошибки, указывающий на тип возникшей ошибки.
</ResponseField>

<ResponseField name="message" type="string">
Человекочитаемое сообщение об ошибке, описывающее, что пошло не так.
</ResponseField>

| Код  | Сообщение                      | Описание    |
| ---- | ------------------------------ | ----------- |
| 4001 | User denied signature request  | Пользователь отклонил запрос на подпись |
| 4100 | Requested method not supported | Метод не поддерживается кошельком |
| 4200 | Wallet not connected           | Отсутствует подключение к кошельку |
| 4300 | Invalid parameters             | Указаны некорректные параметры  |

<Warning>
Всегда проверяйте подписи на стороне сервера, прежде чем доверять им. Проверка подписи на стороне клиента должна использоваться только для улучшения пользовательского опыта (UX).
</Warning>
