## Sign in With Base и Base Pay

Base Account предлагает две кнопки для использования в вашем приложении:
- [**Sign in with Base**](/base-account/reference/ui-elements/sign-in-with-base-button): для аутентификации пользователей в вашем продукте
- [**Base Pay**](/base-account/reference/ui-elements/base-pay-button): для приёма платежей за онлайн- и офлайн-товары

## Sign in with Base

Интеграция "Sign in With Base" предоставляет удобный и надёжный способ для пользователей получать доступ к вашим сервисам. Используя свой существующий Base Account, пользователи могут избежать создания и запоминания новых учётных данных, что обеспечивает более плавный процесс регистрации и входа.

### Рекомендации

Для обеспечения наилучшего пользовательского опыта при интеграции "Sign in With Base" учитывайте следующие рекомендации:

- **Предоставляйте ценность при входе**: Четко сообщайте о преимуществах входа в систему. Пользователи должны понимать, почему их просят войти, например, для персонализации опыта, доступа к премиум-функциям или синхронизации данных между устройствами.

- **Размещайте кнопку заметно**: Сделайте кнопку "Sign in With Base" легко обнаруживаемой. Она не должна быть меньше других вариантов входа и не должна требовать от пользователей прокрутки для её нахождения.

- **Единообразное размещение**: Размещайте кнопку "Sign in With Base" в единообразном и логичном месте на экранах входа и создания аккаунта.

### Дизайн и бренд-гайдлайны

Кнопка "Sign in With Base" должна быть легко узнаваемой и единообразной на всех платформах. Следование этим дизайн-гайдлайнам обеспечивает привычный и доверительный опыт для пользователей.

#### Внешний вид кнопки

Кнопка "Sign in With Base" имеет два ключевых компонента:

1. **Логотип Base — синий квадрат**
   - Квадрат никогда не меняет оттенки синего, он всегда `#0000FF`
   - В тёмном режиме квадрат меняет цвет на чистый белый `#FFFFFF`

2. **Текст "Sign in with Base"**
   - Всегда используйте "Sign in with Base", если только на экране нет явного заголовка "Sign in"
   - По возможности используйте шрифт "Base Sans", в противном случае [создайте пользовательскую кнопку](#creating-a-custom-button)

Ниже представлены рекомендации и ограничения по использованию брендинга Base:

#### РЕКОМЕНДУЕТСЯ
- Оставлять минимум 8 пунктов отступа между синим квадратом Base и текстом "Sign in with Base" при создании пользовательской кнопки
- Использовать синий цвет Base на белом/светлом фоне
- Использовать полностью белую версию на чёрном/тёмном фоне
- Использовать "Sign in with Base" (включая "Sign in"), если только на экране нет заголовка "Sign in"

#### НЕ РЕКОМЕНДУЕТСЯ
- Использовать градиенты для логотипа
- Менять радиус скругления логотипа
- Изменять цвет квадрата Base
- Использовать синий цвет Base на тёмном фоне

### Дизайн и руководства по Бренду

Кнопка "Base Pay" должна быть легко узнаваемой и единообразной на всех платформах. Следование этим дизайн-гайдлайнам обеспечивает привычный и доверительный опыт для пользователей.

#### Внешний вид кнопки

Кнопка "Base Pay" всегда использует комбинированный знак. Она никогда не использует типографику или текст для написания "Base Pay" или "base pay".

Ниже представлены рекомендации и ограничения по использованию брендинга Base:

#### РЕКОМЕНДУЕТСЯ
- Всегда использовать комбинированный знак "Base Pay"
- Использовать полностью белую версию комбинированного знака на тёмных фонах
- Использовать минимум 1X высоты кнопки для отступов. Если высота знака 24px, добавляйте минимум 24px отступов со всех сторон кнопки

#### НЕ РЕКОМЕНДУЕТСЯ
- Писать "Base Pay" или "base pay" с использованием шрифтов или текста
- Изменять комбинированный знак каким-либо образом
- Менять цвет квадрата Base
- Использовать синий цвет Base на тёмном фоне
