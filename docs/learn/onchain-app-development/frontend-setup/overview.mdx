# Обзор

Добро пожаловать! Курс, который вы собираетесь начать, быстро познакомит вас с фронтенд-разработкой для ончейн-приложений и позволит вам создавать веб-сайты, которые могут вызывать функции ваших смарт-контрактов, подобно тому, как традиционные сайты взаимодействуют с API.

## Предварительные требования

Перед началом этих уроков вы должны:

- Уверенно владеть традиционной фронтенд-разработкой с использованием React, желательно с NextJS
- Обладать общим пониманием EVM и смарт-контрактов

---

## Цели

К концу этого курса вы должны уметь:

- **Настройка фронтенда**
  - Определить роль агрегатора кошельков в ончейн-приложении
  - Обсудить плюсы и минусы использования шаблона
  - Создать новое ончейн-приложение с помощью RainbowKit
  - Добавить подключение кошелька в стандартное шаблонное приложение
- **Подключение к блокчейну**
  - Сравнить публичные провайдеры, провайдеры от вендоров и провайдеры кошельков
  - Выбирать подходящий провайдер для различных случаев использования
  - Настроить провайдер в wagmi и использовать его для подключения кошелька
  - Защищать API-ключи, которые будут раскрыты на фронтенде
- **Чтение и отображение данных**
  - Реализовать хук `useAccount` для отображения адреса пользователя, состояния подключения, сети и баланса
  - Реализовать хук `isMounted`, чтобы предотвратить ошибки гидратации (hydration errors)
  - Реализовать хук wagmi `useReadContract` для получения данных из смарт-контракта
  - Преобразовывать данные, полученные из смарт-контракта, в информацию, отображаемую пользователю
  - Определить особенности чтения данных из автоматически сгенерированных геттеров
  - Включить функцию `watch` в `useReadContract` для автоматической выборки обновлений из блокчейна
  - Описать затраты на использование функции `watch` и способы их снижения
  - Настроить аргументы, передаваемые при вызове `pure` или `view` функции смарт-контракта
  - Вызывать экземпляр `useReadContract` по требованию
  - Использовать `isLoading` и `isFetching` для улучшения пользовательского опыта
- **Запись в контракты**
  - Реализовать хук wagmi `useWriteContract` для отправки транзакций в смарт-контракт
  - Настраивать опции в `useWriteContract`
  - Отображать выполнение, успех или неудачу функции с помощью изменения состояния кнопок и отображения данных
  - Реализовать Wagmi `usePrepareContractWrite` и `useWriteContract` для отправки транзакций в смарт-контракт
  - Настраивать опции в `useSimulateContract` и `useWriteContract`
  - Вызывать функцию смарт-контракта по требованию, используя функцию write из `useWriteContract`, с аргументами и значением (value)
