Хотя блокчейны по умолчанию публично отображают адреса кошельков, эти длинные строки неудобны для повседневного использования. Ончейн-системы именования, такие как Basenames, решают эту проблему, позволяя пользователям регистрировать читаемые имена (например, "alice.base"). Помимо простого именования, ончейн-идентичность может быть расширена с помощью верификаций и атестаций, которые подтверждают дополнительные факты (например, что пользователь является верифицированным владельцем аккаунта или проживает в определенной стране). Здесь вы узнаете, как Basenames и ончейн-верификации работают вместе, чтобы сделать ваше приложение более удобным и заслуживающим доверия.

## Почему важна идентичность

<Info>
**Читаемость адресов**: По умолчанию адреса кошельков представляют собой длинные, нечитаемые hex-строки. Ончейн-имена (например, "alice.base") упрощают идентификацию получателей или пользователей.
</Info>

<Tip>
**Доверие без риска для конфиденциальности**: Атестации/Верификации позволяют вашему приложению доверять определенным атрибутам пользователя - без раскрытия личных данных вне цепи.
</Tip>

## Basenames

<Check>
**Человеко-читаемые имена** для адресов в сети Base (аналогично DNS для IP-адресов)
</Check>

<Check>
**Унифицированная идентичность**: Пользователи могут регистрировать имена, добавлять изображение профиля и отображать их во всех приложениях, к которым они подключаются
</Check>

<Check>
**Снижение трения**: Помогает унифицировать ончейн-идентичность и снизить трение для пользователей
</Check>

## Ончейн-верификации

<Info>
**Проблема**: Цепь не может изначально видеть данные вне цепи (например, ваше гражданство или статус членства).
</Info>

<Tip>
**Решение**: Верификации позволяют вам публиковать атестации **в ончейн-среде**, чтобы другие контракты могли их читать без риска для конфиденциальности. Например, подтвердите, что вы пользователь Coinbase или резидент определенной страны, и автоматически разблокируйте особые привилегии в децентрализованных приложениях (dapps).
</Tip>
