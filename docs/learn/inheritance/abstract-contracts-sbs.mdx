[Abstract] контракты не могут существовать сами по себе. Их функциональность может быть использована только контрактом, который наследует от них. В этом уроке вы узнаете, как создать абстрактный контракт.

---

## Цели

К концу этого урока вы сможете:

-Использовать ключевые слова virtual, override и abstract для создания и использования абстрактного контракта

---

## Абстрактные контракты

Продолжайте работу с вашим файлом `Inheritance.sol`. Добавьте `ContractD` как `abstract contract`. Добавьте `virtual` функцию с именем `whoAreYou` function, но **не** добавляйте никакой реализации для этой функции.

<Accordion title="Показать код">

```solidity
abstract contract ContractD {
    function whoAreYou() public virtual view returns (string memory);
}
```

</Accordion>

### Наследование от абстрактной функции

Обновите `ContractA`, чтобы он наследовал от `ContractD`.

Вы получите несколько запутанную ошибку о том, что `ContractA` должен быть помечен как `abstract`. Сделать это - **не** правильное исправление.

```text
from solidity:
TypeError: Contract "ContractA" should be marked as abstract.
  --> contracts/Inheritance.sol:25:1:
   |
25 | contract ContractA is ContractB, ContractC, ContractD {
   | ^ (Relevant source part starts here and spans across multiple lines).
Note: Missing implementation:
 --> contracts/Inheritance.sol:6:5:
  |
6 |     function whoAreYou() public virtual view returns (string memory);
  |     ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
```

Подсказка для правильного решения находится ниже: `Note:  Missing implementation:`

Только `abstract` контракты могут объявлять функции без реализации. Чтобы исправить это, предоставьте `override` реализацию для `whoAreYou` в `ContractA`:

<Accordion title="Показать код">

```solidity
function whoAreYou() public override pure returns (string memory) {
    return "I'm a person!";
}
```

</Accordion>
