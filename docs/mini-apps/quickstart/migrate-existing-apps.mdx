**Предварительные требования** 
* У вас есть существующее веб-приложение
* У вас есть аккаунт Base app


<Steps>
<Step title="Add the MiniApp SDK">
<CodeGroup>
```bash npm
npm install @farcaster/miniapp-sdk
```

```bash pnpm
pnpm add @farcaster/miniapp-sdk
```

```bash yarn
yarn add @farcaster/miniapp-sdk
```
</CodeGroup>
</Step>

<Step title="Запустите отображение приложения">
После загрузки вашего приложения вызовите `sdk.actions.ready()`, чтобы скрыть загрузочный экран и отобразить ваше приложение.
<Tabs>
    <Tab title="Vanilla JS">
    ```javascript app.js
    import { sdk } from '@farcaster/miniapp-sdk';
    
    // Как только приложение готово к отображению
    await sdk.actions.ready();
    ```
    </Tab>
    <Tab title="React">
   В React-приложениях вызывайте `ready()` внутри хука `useEffect`, чтобы предотвратить его выполнение при каждом повторном рендере. Вызывайте `ready()` как можно скорее и избегайте дрожания и         перестроения контента.
    ```typescript app.tsx
    import { sdk } from '@farcaster/miniapp-sdk';
    import { useEffect } from 'react';
    
    function App() {
        useEffect(() => {
            sdk.actions.ready();
        }, []);

        return(...your app content goes here...)
    }

    export default App;
    
    ```
    </Tab>
</Tabs>
</Step>

<Step title="Разместите манифест">
Создайте файл, доступный по адресу `https://www.your-domain.com/.well-known/farcaster.json`.

<Tabs>
    <Tab title="Vanilla JS">
   Создайте файл манифеста в вашем проекте по пути `/public/.well-known/farcaster.json`.
    </Tab>
    <Tab title="Next.js">
    Создайте Next.js-роут для размещения вашего файла манифеста
    ```typescript app/.well-known/farcaster.json/route.ts
    function withValidProperties(properties: Record<string, undefined | string | string[]>) {
    return Object.fromEntries(
        Object.entries(properties).filter(([_, value]) => (Array.isArray(value) ? value.length > 0 : !!value))
    );
    }

    export async function GET() {
    const URL = process.env.NEXT_PUBLIC_URL as string;
    return Response.json(paste_manifest_json_object_here); // см. следующий шаг для объекта manifest_json_object
    }
    ```
    </Tab>
</Tabs>

</Step>

<Step title="Создайте учетные данные accountAssociation">
Поля `accountAssociation` в манифесте используются для проверки владения вашим приложением. Вы можете сгенерировать эти поля в Base Build.


    1. Убедитесь, что все изменения залиты в продакшн, чтобы файл манифеста был доступен по URL вашего приложения.
    2. Перейдите в [инструмент связывания аккаунтов](https://www.base.dev/preview?tab=account) в Base Build.
    3. Вставьте ваш домен в поле `App URL` (например, sample-url.vercel.app) и нажмите "Submit"
    4. Нажмите кнопку "Verify", которая появится, и следуйте инструкциям для генерации полей `accountAssociation`.
    5. Скопируйте поля `accountAssociation` и вставьте их в файл манифеста, который вы добавили на предыдущем шаге.

<Step title="Добавьте метаданные для встраивания">
  Обновите ваш файл index.html, чтобы включить метаданные `fc:miniapp`. Они используются для создания богатых эмбедов при публикации вашего приложения и необходимы для отображения вашего приложения.

  <Tabs>
    <Tab title="Vanilla JS">
    Добавьте напрямую в ваш файл index.html.
  ```html index.html
    <meta name="fc:miniapp" content='{
    "version":"next",
    "imageUrl":"https://your-app.com/embed-image",
    "button":{
        "title":"Play Now",
        "action":{
        "type":"launch_miniapp",
        "name":"Your App Name",
        "url":"https://your-app.com"
        }
    }
    }' />
    ```
    </Tab>
    <Tab title="Next.js">
    Используйте функцию `generateMetadata` для добавления метаданных `fc:miniapp`.
    ```typescript app/layout.tsx
        export async function generateMetadata(): Promise<Metadata> {
        return {
            other: {
            'fc:miniapp': JSON.stringify({
                version: 'next',
                imageUrl: 'https://your-app.com/embed-image',
                button: {
                    title: `Launch Your App Name`,
                    action: {
                        type: 'launch_miniapp',
                        name: 'Your App Name',
                        url: 'https://your-app.com',
                        splashImageUrl: 'https://your-app.com/splash-image',
                        splashBackgroundColor: '#000000',
                    },
                },
            }),
            },
        };
        }
    ```
    </Tab>
</Tabs>
</Step>
<Step title="Выгрузите в продакшн">
  Убедитесь, что все изменения залиты в продакшн.
</Step>

<Step title="Просмотрите ваше приложение">
  Используйте [инструмент предпросмотра](https://www.base.dev/preview) в Base Build для проверки вашего приложения.
  1. Добавьте URL вашего приложения для просмотра эмбедов и нажмите кнопку запуска, чтобы убедиться, что приложение запускается ожидаемым образом.
  2. Используйте вкладку "Account association", чтобы проверить правильность создания учетных данных для связывания аккаунтов.
  3. Используйте вкладку "Metadata", чтобы увидеть метаданные, добавленные из манифеста, и определить отсутствующие поля.
