<Steps>
<Step title="Разверните шаблон">
Нажмите кнопку ниже и следуйте инструкциям, чтобы развернуть шаблон быстрого старта в Vercel.
<Card title="Развернуть в Vercel" icon="rocket" href="https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fbase%2Fdemos%2Ftree%2Fmaster%2Fmini-apps%2Ftemplates%2Fminikit%2Fnew-mini-app-quickstart&project-name=new-mini-app-quickstart&repository-name=new-mini-app-quickstart&env=NEXT_PUBLIC_PROJECT_NAME&demo-title=New%20Mini%20App&demo-description=Quickstart%20waitlist%20mini%20app%20with%20MiniKit%20on%20Base" horizontal>
  Быстро разверните шаблон быстрого старта в Vercel, чтобы начать работу.
</Card>
</Step>
<Step title="Клонируйте ваш репозиторий">
Клонируйте репозиторий, созданный Vercel, чтобы вносить локальные изменения.

Замените `<your-username>` на ваше имя пользователя GitHub.
```bash Terminal
git clone https://github.com/<your-username>/new-mini-app-quickstart
cd new-mini-app-quickstart
npm install
```
</Step>
<Step title="Обновите конфигурацию манифеста">
Файл `minikit.config.ts` отвечает за настройку вашего манифеста, расположенного по адресу `app/.well-known/farcaster.json`, и создание метаданных для встраивания.

<Tip> Вы можете настроить манифест, обновив объект `miniapp`.</Tip>

Подробности о каждом поле см. в [справочнике полей](/mini-apps/features/manifest#field-reference).
```ts minikit.config.ts
export const minikitConfig = {
  accountAssociation: { // это будет добавлено в шаге 5
    "header": "",
    "payload": "",
    "signature": ""
  },
  miniapp: {
    version: "1",
    name: "Cubey", 
    subtitle: "Your AI Ad Companion", 
    description: "Ads",
    screenshotUrls: [`${ROOT_URL}/screenshot-portrait.png`],
    iconUrl: `${ROOT_URL}/blue-icon.png`,
    splashImageUrl: `${ROOT_URL}/blue-hero.png`,
    splashBackgroundColor: "#000000",
    homeUrl: ROOT_URL,
    webhookUrl: `${ROOT_URL}/api/webhook`,
    primaryCategory: "social",
    tags: ["marketing", "ads", "quickstart", "waitlist"],
    heroImageUrl: `${ROOT_URL}/blue-hero.png`, 
    tagline: "",
    ogTitle: "",
    ogDescription: "",
    ogImageUrl: `${ROOT_URL}/blue-hero.png`,
  },
} as const;
```

</Step>

<Step title="Создайте учетные данные accountAssociation">

Теперь, когда у вас есть публичный домен для вашего приложения, вы готовы связать ваше мини-приложение с вашим аккаунтом Farcaster.


    1. Убедитесь, что все изменения залиты в продакшн, отправив изменения в ветку `main`.
    <Tip>Убедитесь, что **Deployment Protection** в Vercel выключен, перейдя в панель управления Vercel для вашего проекта и перейдя в Settings -> Deployment Protection, переключив "Vercel Authentication" в положение off и нажав save. </Tip>

    2. NПерейдите в [инструмент связывания аккаунтов](https://www.base.dev/preview?tab=account) в Base Build.
    3. Вставьте ваш домен в поле `App URL` (например, sample-url.vercel.app) и нажмите "Submit"

    <img 
      src="/images/base-build/sign-manifest.png" 
      alt="Sign manifest"
      height="300"
      className="rounded-lg"  
    /> 

    4. Нажмите кнопку "Verify", которая появится, и следуйте инструкциям для генерации полей `accountAssociation`.
    5. Скопируйте объект `accountAssociation`

</Step>

<Step title="Обновите `minikit.config.ts`">
Обновите ваш файл `minikit.config.ts`, чтобы включить объект `accountAssociation`, который вы скопировали на предыдущем шаге.

```ts minikit.config.ts
export const minikitConfig = {
    accountAssociation: {
        "header": "eyJmaBBiOjE3MzE4LCJ0eXBlIjoiY3VzdG9keSIsImtleSI6IjB4NzYwQjA0NDc5NjM4MTExNzNmRjg3YDPBYzA5OEJBQ0YxNzNCYkU0OCJ9",
        "payload": "eyJkb21haW4iOiJ4BWl0bGlzdC1xcy52ZXJjZWwuYXBwIn7",
        "signature": "MHhmNGQzN2M2OTk4NDIwZDNjZWVjYTNiODllYzJkMjAwOTkyMDEwOGVhNTFlYWI3NjAyN2QyMmM1MDVhNzIyMWY2NTRiYmRlZmQ0NGQwOWNiY2M2NmI2B7VmNGZiMmZiOGYzNDVjODVmNmQ3ZTVjNzI3OWNmMGY4ZTA2ODYzM2FjZjFi"
    },
    miniapp: {
        ...
    },
  }
```
</Step>
<Step title="Отправьте обновления в продакшн">
Отправьте все изменения в ветку `main`. Vercel автоматически развернёт изменения в вашу продакшен-среду.
</Step>

<Step title="Просмотрите ваше приложение">
  Перейдите на [base.dev/preview](https://base.dev/preview) для проверки вашего приложения.
  1. Добавьте URL вашего приложения для просмотра эмбедов и нажмите кнопку запуска, чтобы убедиться, что приложение запускается ожидаемым образом.
  2. Используйте вкладку "Account association", чтобы проверить правильность создания учетных данных для связывания аккаунтов.
  3. Используйте вкладку "Metadata", чтобы увидеть метаданные, добавленные из манифеста, и определить отсутствующие поля.
