## Реализация

1. Создайте файл манифеста в вашем проекте по пути `/public/.well-known/farcaster.json`. Он должен быть доступен по адресу `https://your-domain.com/.well-known/farcaster.json`
2. Обновите [обязательные](#accountassociation) и [опциональные](#display-information) поля в объекте `miniapp`
3. Убедитесь, что все изменения залиты в продакшн, чтобы файл манифеста был доступен по URL вашего приложения
4. Перейдите в [инструмент связывания аккаунтов](https://www.base.dev/preview?tab=account) в Base Build
5. Вставьте ваш домен в поле App URL (например, sample-url.vercel.app) и нажмите "Submit"
6. Нажмите кнопку "Verify", которая появится, и подпишите манифест вашим кошельком для генерации полей `accountAssociation`
7. Скопируйте сгенерированные поля `accountAssociation` (header, payload и signature) и вставьте их в ваш файл манифеста, заменив пустые значения в объекте `accountAssociation`

<Warning>
Изменения в манифесте вступают в силу после повторного деплоя и повторной публикации вашего Mini App. Платформа повторно индексирует обновлённую конфигурацию и применяет изменения к поиску, отображению в каталоге и рендерингу встроенных элементов (embeds).
</Warning>

<Tip>
Используйте [Генератор Mini App Assets](https://www.miniappassets.com/) для создания правильно отформатированных иконок, загрузочных экранов и изображений, соответствующих требованиям как Base App, так и мини-приложений Farcaster.
</Tip>




## Схема

### accountAssociation

Доказывает владение доменом для вашего мини-приложения.
<Card>
<ParamField path="header" type="string" required>
Кодированный заголовок для полезной нагрузки связывания.
</ParamField>

<ParamField path="payload" type="string" required>
Кодированная полезная нагрузка, содержащая ваш домен.
</ParamField>

<ParamField path="signature" type="string" required>
Подпись над полезной нагрузкой.
</ParamField>
</Card>

### baseBuilder

Проверяет право собственности и связывает ваш аккаунт Base Build. Этот адрес должен быть адресом кошелька, использованного при импорте вашего мини-приложения в Base Build.
<Card>
<ParamField path="ownerAddress" type="string" required>
Проверяет право собственности и связывает ваш аккаунт Base Build.
</ParamField>
</Card>


#### Идентичность и запуск

Определяет основную идентичность вашего мини-приложения и URL, на который попадают пользователи при его открытии.

<Card>
<ParamField path="version" type="string" required>
Версия манифеста. Должна быть `"1"`.
</ParamField>

<ParamField path="name" type="string" required>
Название мини-приложения. Макс. 32 символа.
</ParamField>

<ParamField path="homeUrl" type="string" required>
URL запуска по умолчанию. HTTPS URL, макс. 1024 символа.
</ParamField>

<ParamField path="iconUrl" type="string" required>
URL изображения иконки. HTTPS URL, PNG 1024×1024; прозрачный фон не рекомендуется.
</ParamField>
</Card>

#### Опыт загрузки

Управляет визуальными элементами и цветами экрана-заставки, показываемыми во время загрузки вашего мини-приложения.

<Card>
<ParamField path="splashImageUrl" type="string" required>
Изображение загрузки. HTTPS URL, рекомендуется 200×200px.
</ParamField>

<ParamField path="splashBackgroundColor" type="string" required>
Цвет фона загрузки. Hex-код (например, `#000000`).
</ParamField>
</Card>

#### Обнаружение и поиск

Определяет, как ваше мини-приложение индексируется, категоризируется и появляется в функциях обнаружения Base App.

<Card>
<ParamField path="primaryCategory" type="string" required>
Определяет, где ваше приложение появляется в просмотре по категориям. Одно из: `games`, `social`, `finance`, `utility`, `productivity`, `health-fitness`, `news-media`, `music`, `shopping`, `education`, `developer-tools`, `entertainment`, `art-creativity`.
</ParamField>

<ParamField path="tags" type="string[]" required>
Теги для поиска/фильтрации. До 5; ≤ 20 символов каждый; строчные буквы; без пробелов/эмодзи/специальных символов.
</ParamField>

<ParamField path="noindex" type="boolean">
Исключить из результатов поиска. `true` = исключить, по умолчанию = включать.
</ParamField>
</Card>

#### Информация для отображения

Предоставляет описательный текст, скриншоты и промо-изображения, показываемые в профиле вашего мини-приложения.

<Card>
<ParamField path="subtitle" type="string">
Короткое описание под названием. Макс. 30 символов; избегайте эмодзи/специальных символов.
</ParamField>

<ParamField path="description" type="string">
Промо-текст для страницы приложения. Макс. 170 символов; избегайте эмодзи/специальных символов.
</ParamField>

<ParamField path="tagline" type="string">
Маркетинговый слоган. Макс. 30 символов.
</ParamField>

<ParamField path="heroImageUrl" type="string">
Большое промо-изображение. 1200×630px (1.91:1), PNG/JPG.
</ParamField>

<ParamField path="screenshotUrls" type="string[]">
Визуальные превью. Макс. 3; рекомендуется портретное соотношение 1284×2778px.
</ParamField>
</Card>

#### Уведомления

Конечная точка для уведомлений.
<Card>
<ParamField path="webhookUrl" type="string">
Конечная точка для POST-событий. HTTPS URL, макс. 1024 символа. Требуется при использовании уведомлений.
</ParamField>
</Card>
