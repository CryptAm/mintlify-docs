## Полные метаданные

Метаданные включают ваш манифест и метаданные для встраивания. Полные, валидные метаданные необходимы для индексации, размещения в категориях и создания качественных встраиваний.

**Критерии приемки**
- Манифест общедоступен по адресу `/.well-known/farcaster.json` 
- Обязательные поля присутствуют и валидны (`accountAssociation`, `frame`, `primaryCategory`, `tags`)
- Изображения соответствуют ограничениям по размеру/формату; текстовые поля соблюдают ограничения по длине

**Как реализовать**
- Следуйте [руководству по манифесту](/mini-apps/core-concepts/manifest)
- Реализуйте [метаданные для встраиваний](/mini-apps/core-concepts/embeds-and-previews#implementation)

<Note>
  Проверьте свой манифест с помощью наших инструментов предпросмотра на <a href="https://base.dev/preview">base.dev/preview</a>.
</Note>

## Внутриприложенная аутентификация

Пользователи должны оставаться в приложении Base на протяжении всего процесса аутентификации. Исключите потоки, которые выводят пользователей из приложения Base.

**Критерии приемки**
- Нет внешних перенаправлений
- Нет подтверждения по электронной почте / телефону
- По возможности пользователи могут изучать приложение до входа в систему

**Как реализовать**
- Следуйте [руководству по аутентификации](/mini-apps/core-concepts/authentication)
- Предпочитайте внутриприложенную SIWF/Quick Auth или аутентификацию через кошелек;

## Клиент-агностичность

Не должно быть поведения или формулировок, специфичных для какого-либо клиента, которые ухудшают опыт в приложении Base. Вы также должны гарантировать, что не перенаправляете пользователя в другой клиент для функций, поддерживаемых в приложении Base.

**Критерии приемки**
- Не используйте жестко закодированные URL, специфичные для клиента (например, ссылки, работающие только в Farcaster)
- Используйте нейтральный язык в пользовательском интерфейсе (например, "Поделиться в ленте" вместо "Поделиться в Farcaster")
- Устраните кнопки, которые используют диплинкинг в другие клиенты для функций, поддерживаемых в приложении Base

**Как реализовать**
- Обновите все ссылки в соответствии с [руководством по ссылкам](/mini-apps/technical-guides/links)
- Ознакомьтесь с руководством [Совместимость с Base App](/mini-apps/troubleshooting/base-app-compatibility) для функциональности, не поддерживаемой в приложении Base. Вся остальная функциональность должна удерживать пользователей в приложении Base.

## Спонсирование транзакций

Спонсируйте комиссии за транзакции, чтобы снизить трение и уменьшить отток новых пользователей. Для мини-приложений на Base мы рекомендуем использовать [Base Paymaster](/onchainkit/paymaster/quickstart-guide).

**Критерии приемки**
- Транзакции спонсируются через paymaster

**Как реализовать**
- Рекомендуется: [Base Paymaster](/onchainkit/paymaster/quickstart-guide)

<Note>
  Получите бесплатные газовые кредиты на <a href="https://base.dev">base.dev</a>.
</Note>


## Пакетные транзакции (EIP-5792)

Объединяйте последовательные действия там, где это применимо, чтобы минимизировать количество подписей и снизить трение. Используйте возможности EIP‑5792 для отправки нескольких вызовов в одном запросе.

**Критерии приемки**
- Там, где это применимо, объединяйте последовательные действия в один пакет (например, approve + swap)

**Как реализовать**
- Смотрите [Пакетные транзакции](/base-account/improve-ux/batch-transactions)
- API провайдера: [`wallet_sendCalls`](/base-account/reference/core/provider-rpc-methods/wallet_sendCalls), [`wallet_getCapabilities`](/base-account/reference/core/provider-rpc-methods/wallet_getCapabilities)
