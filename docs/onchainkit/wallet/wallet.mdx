Компоненты `<Wallet />` предоставляют интерфейс для пользователей, чтобы подключить
их смарт-кошелек с информацией об идентификации, такой как Basename и баланс ETH.

Они оснащены встроенными отполированными пользовательскими интерфейсами как для веб,
так и для мобильного веба, что делает их невероятно простыми для улучшения с помощью компонентов.

Перед их использованием убедитесь, что вы выполнили все [Getting Started steps](/onchainkit/getting-started).

## Быстрый старт

Чтобы упростить процесс интеграции и внедрить упрощенную версию компонента `Wallet`, вы можете отобразить компонент `Wallet` без дочерних элементов.

Если вам требуется больше кастомизации, следуйте руководству по реализации для нашего компонента `Wallet` ниже, где вы можете вручную определить каждый дочерний компонент и свойство.

```tsx
import { Wallet } from '@coinbase/onchainkit/wallet';

<Wallet />
```

## Пошаговое руководство

<Steps>
  <Step title="Настройка подключений кошелька">
  Начните настройку подключения кошелька, сконфигурировав провайдер `wagmi`.

  И не забудьте обновить `appName`, так как это имя будет показано 
  пользователю при подключении кошелька.

  ```tsx
  // @noErrors: 2554
  import { ReactNode } from 'react';
  import { WagmiProvider, createConfig, http } from 'wagmi';
  import { baseSepolia } from 'wagmi/chains';
  import { coinbaseWallet } from 'wagmi/connectors';

  const wagmiConfig = createConfig({
    chains: [baseSepolia],
    connectors: [
      coinbaseWallet({
        appName: 'onchainkit',
      }),
    ],
    ssr: true,
    transports: {
      [baseSepolia.id]: http(),
    },
  });

  function App({ children }: { children: ReactNode }) {
    return <WagmiProvider config={wagmiConfig}>{children}</WagmiProvider>;
  }
  ```

  </Step>
  <Step title="Добавление компонентов <Wallet />">
  Испытайте магию, просто вставив компонент `<Wallet />`, и наблюдайте,
  как он плавно оживает.

  Кроме того, вы можете видеть, что компоненты [`<Identity>`](/onchainkit/identity/identity)
  такие как [`<Avatar>`](/onchainkit/identity/avatar), [`<Name>`](/onchainkit/identity/name),
  и [`<Address>`](/onchainkit/identity/address) используются композиционным способом.
  Изучите их страницы документации, чтобы открыть для себя различные варианты настройки.

  ```tsx
  import {
    ConnectWallet,
    Wallet,
    WalletDropdown,
    WalletDropdownDisconnect,
  } from '@coinbase/onchainkit/wallet';
  import {
    Address,
    Avatar,
    Name,
    Identity,
  } from '@coinbase/onchainkit/identity';
  import { color } from '@coinbase/onchainkit/theme';

  export function WalletComponents() {
    return (
      <div className="flex justify-end">
        <Wallet>
          <ConnectWallet>
            <Avatar className="h-6 w-6" />
            <Name />
          </ConnectWallet>
          <WalletDropdown>
            <Identity className="px-4 pt-3 pb-2" hasCopyAddressOnClick>
              <Avatar />
              <Name />
              <Address className={color.foregroundMuted} />
            </Identity>
            <WalletDropdownDisconnect />
          </WalletDropdown>
        </Wallet>
      </div>
    );
  }
  ```

## Кастомизация текста и стиля кнопки подключения

Каждый компонент OnchainKit предлагает гибкость для настройки `className`
и изменения стиля представляемых им React-компонентов.

Настройте текст подключения кошелька, используя свойство `disconnectedLabel` в компоненте `<ConnectWallet />`.

```tsx
import {
  ConnectWallet,
  Wallet,
  WalletDropdown,
  WalletDropdownLink,
  WalletDropdownDisconnect,
} from '@coinbase/onchainkit/wallet';
import {
  Address,
  Avatar,
  Name,
  Identity,
  EthBalance,
} from '@coinbase/onchainkit/identity';
import { color } from '@coinbase/onchainkit/theme';
// ---cut-before---
// опущено для краткости

<Wallet>
  <ConnectWallet className='bg-blue-800' disconnectedLabel='Log In'>
    <Avatar className="h-6 w-6" />
    <Name className='text-white' />
  </ConnectWallet>
  <WalletDropdown>
    <Identity
      className="px-4 pt-3 pb-2 hover:bg-blue-200"
      hasCopyAddressOnClick
    >
      <Avatar />
      <Name />
      <Address />
      <EthBalance />
    </Identity>
    <WalletDropdownLink
      className='hover:bg-blue-200'
      icon="wallet"
      href="https://keys.coinbase.com"
    >
      Wallet
    </WalletDropdownLink>
    <WalletDropdownDisconnect className='hover:bg-blue-200' />
  </WalletDropdown>
</Wallet>
```

## Пример использования

### Использование с Sign In With Ethereum (SIWE)

Чтобы использовать [Sign In With Ethereum (SIWE)](https://docs.login.xyz/general-information/siwe-overview) с OnchainKit, вы можете использовать свойство `onConnect` в компоненте `<ConnectWallet />`. Это вызовет запрос SIWE, когда пользователь подключит свой кошелек.

```tsx
import { ConnectWallet } from '@coinbase/onchainkit/wallet';
import { base } from 'wagmi/chains';
import { createSiweMessage } from 'viem/siwe'
import { useSignMessage } from 'wagmi';

const message = createSiweMessage({
  address: '0xA0Cf798816D4b9b9866b5330EEa46a18382f251e',
  chainId: base.id,
  domain: 'example.com',
  nonce: 'foobarbaz',
  uri: 'https://example.com/path',
  version: '1',
})

export function WalletComponents() {
  const { signMessage } = useSignMessage();

  return (
    <ConnectWallet onConnect={() => {signMessage({ message })}} />
  );
}

```

## Компоненты

Компоненты предназначены для совместной иерархической работы. Для каждого компонента убедитесь в следующем:

- `<Wallet />` - Служит основным контейнером для всех компонентов, связанных с кошельком.
- `<ConnectWallet />` - Обрабатывает процесс подключения кошелька. Размещайте дочерние компоненты внутри, чтобы настроить внешний вид кнопки подключения.
- `<AppWithWalletModal />` - Включает опыт агрегации кошельков.
- `<WalletDropdown />` - Содержит дополнительную информацию и опции кошелька. Размещайте внутри компонента `<Wallet />`.
- `<Identity />` - Отображает информацию об идентификации пользователя. Размещайте внутри `<WalletDropdown />` для полного представления профиля.
- `<WalletDropdownBasename />` - Отображает Basename пользователя внутри выпадающего списка.
- `<WalletDropdownLink />` - Создает пользовательскую ссылку внутри выпадающего списка. Используйте свойство `icon` для добавления иконки и `href` для указания назначения.
- `<WalletDropdownDisconnect />` - Предоставляет опцию отключения внутри выпадающего списка.

Дополнительные компоненты для настройки интерфейса кошелька включают:

- `<Avatar />` - Отображает изображение аватара пользователя.
- `<Name />` - Показывает имя пользователя или ENS.
- `<Badge />` - Может использоваться для отображения дополнительного статуса или информации о пользователе.
- `<Address />` - Показывает адрес кошелька пользователя.
- `<EthBalance />` - Отображает баланс ETH пользователя.

Компонент Wallet автоматически обрабатывает состояние подключения кошелька и соответствующим образом обновляет пользовательский интерфейс.
Вам нужно обернуть ваше приложение или соответствующую его часть этими компонентами,
чтобы обеспечить полноценный опыт взаимодействия с кошельком.
