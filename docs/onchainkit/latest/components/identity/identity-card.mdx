Компонент `IdentityCard` предоставляет комплексный способ отображения информации об идентичности пользователя, включая ENS имена, аватары и разрешение имен для конкретной цепи.

## Особенности
- **Разрешение имен:** Автоматически разрешает как Basenames, так и ENS имена
- **Поддержка аватаров:** Отображает аватары ENS и специфичные для цепи
- **Гибкое отображение:** Настраиваемые варианты макета и стилей
- **Осведомленность о цепи:** Работает с разными EVM-цепями, поддерживающими разрешение имен
- **Поддержка подсказок (Tooltip):** Отображает информацию об аттестации при наведении на бейдж

## Кастомизация

Вы можете переопределить стили, используя `className` или установив кастомную [OnchainKit theme](/onchainkit/guides/themes#custom-theme). Вы также можете установить цепь `mainnet` для разрешения ENS имен:

```tsx
import { IdentityCard } from '@coinbase/onchainkit/identity';
import { mainnet } from 'viem/chains'; // [!code focus]

<OnchainKitProvider
  config={{
    appearance: {
      mode: 'auto',
      theme: 'cyberpunk', // [!code focus]
    },
  }}
>
</OnchainKitProvider>

<IdentityCard 
  address="0x123..."
  chain={mainnet} // [!code focus]
  schemaId="0xf8b05c79f090979bf4a80270aba232dff11a10d9ca55c4f88de95317970f0de9"
/>
```

## Свойства

[`IdentityCardReact`](/onchainkit/identity/types#identitycardreact)

| Свойство | Тип  | Описание    |
|----------|------|-------------|
| `address` | `string` | Адрес кошелька для отображения идентичности |
| `chain` | `Chain` | Цепь, на которой разрешать идентичность |
| `className` | `string` | Дополнительные CSS-классы для применения |
| `schemaId` | `Address \| null` | ID схемы для аттестации |
| `badgeTooltip` | `boolean \| string` | При `true` отображает название аттестации в подсказке. При передаче строки показывает этот кастомный текст. По умолчанию `false` |

## Обработка ошибок

- Компонент корректно обрабатывает различные состояния ошибок:
- Невалидные адреса отображаются в сокращенном формате
- Отсутствие ENS имен приводит к откату на сокращенные адреса
- Неудачная загрузка аватара показывает аватар по умолчанию
- Ошибки сети поддерживают деградированное, но функциональное отображение

## Рекомендации

1. Всегда предоставляйте валидный объект цепи из viem/chains
2. Обрабатывайте состояния загрузки в родительских компонентах, когда адрес может быть неопределен
3. Реализуйте правильные границы ошибок в родительских компонентах
4. Учитывайте адаптивность для мобильных устройств при стилизации
5. Используйте `badgeTooltip`, чтобы предоставить контекст о том, что представляет собой бейдж верификации
