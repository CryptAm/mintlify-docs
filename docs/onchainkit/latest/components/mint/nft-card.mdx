Компонент `NFTCard` предоставляет простой способ просмотра любого NFT. Просто введите адрес контракта NFT и идентификатор токена и включите дочерние компоненты, которые вы хотите отобразить.

### Создание нового проекта

Если вы начинаете новый проект, мы рекомендуем использовать `create onchain` для создания структуры вашего проекта.

```bash
npm create onchain@latest
```

### Добавление в существующий проект

Если вы добавляете `NFTCard` в существующий проект, просто установите OnchainKit.

<CodeGroup>
```bash npm
npm install @coinbase/onchainkit
```

```bash yarn
yarn add @coinbase/onchainkit
```

```bash pnpm
pnpm add @coinbase/onchainkit
```

```bash bun
bun add @coinbase/onchainkit
```
</CodeGroup>

Оберните `<OnchainKitProvider />` вокруг вашего приложения, следуя шагам в [Getting Started](/onchainkit/getting-started#add-providers).

## Быстрый старт

Компонент `NFTCardDefault` является упрощенной версией компонента `NFTCard`, предназначенной для упрощения процесса интеграции для разработчиков. Вместо ручного определения каждого подкомпонента разработчики могут использовать эту сокращенную версию, которая отображает нашу рекомендуемую реализацию компонента и включает `NFTTitle`, `NFTMedia`, `NFTOwner`, `NFTLastSoldPrice` и `NFTNetwork`.

```tsx
import { NFTCardDefault } from '@coinbase/onchainkit/nft'; // [!code focus]

<NFTCardDefault // [!code focus]
  contractAddress='0xb4703a3a73aec16e764cbd210b0fde9efdab8941'  // [!code focus]
  tokenId='1' // [!code focus]
/>  // [!code focus]
```

Если вы хотите больше кастомизации, вы можете выполнить следующие шаги, чтобы настроить, какие подкомпоненты отображаются.

## Расширенное использование

### Используйте свои собственные данные

Реализация `NFTCard` по умолчанию использует Coinbase Developer Platform для предоставления данных и требует API-ключа. Если вы хотите использовать свои собственные данные, вы можете использовать свойство `useNFTData` для передачи хука, который получает данные NFT из источника по вашему выбору. Пока этот хук возвращает объект с той же структурой, что и тип [`NFTData`](/onchainkit/mint/types#nftdata), NFTCard будет отображаться.

```tsx
import { NFTCard } from '@coinbase/onchainkit/nft';
import {
  NFTMedia,
  NFTTitle,
} from '@coinbase/onchainkit/nft/view';
// ---cut-before---

function useNFTData() {  // [!code focus]
  return {  // [!code focus]
    title: 'My NFT',  // [!code focus]
    imageUrl: 'https://example.com/image.png',  // [!code focus]
  }  // [!code focus]
}  // [!code focus]

export default function NFTComponent() {
  return (
    <NFTCard
      contractAddress='0xb4703a3a73aec16e764cbd210b0fde9efdab8941'
      tokenId='1'
      useNFTData={useNFTData}  // [!code focus]
    >
      <NFTMedia/>
      <NFTTitle />
    </NFTCard>
  );
}
```

### Отслеживание жизненного цикла компонента

Вы можете использовать наш NFT [`LifecycleStatus`](/onchainkit/mint/types#lifecyclestatus) и свойство `onStatus` для отслеживания состояний транзакции.

NFTCard имеет 4 состояния жизненного цикла:
* `init` - Компонент инициализирован
* `error` - Компонент столкнулся с ошибкой
* `mediaLoading` - Медиа загружается
* `success` - Медиа загружено

```tsx
import { NFTCard } from '@coinbase/onchainkit/nft';
import { NFTMedia } from '@coinbase/onchainkit/nft/view';
// ---cut-before---
import type { LifecycleStatus } from '@coinbase/onchainkit/nft'; // [!code focus]

const statusHandler = (status: LifecycleStatus) => { // [!code focus]
  const { statusName, statusData } = status; // [!code focus]
  switch (statusName) { // [!code focus]
    case 'success': // [!code focus]
      // обработка успеха
    case 'error': // [!code focus]
      // обработка ошибки
    default: // [!code focus]
      // обработка состояния 'init'
  } // [!code focus]
} // [!code focus]

<NFTCard  // [!code focus]
  onStatus={statusHandler}  // [!code focus]
  contractAddress='0xb4703a3a73aec16e764cbd210b0fde9efdab8941'
  tokenId='1'
>
  <NFTMedia />
</NFTCard>
// ---cut-after---
```

## Примеры использования
- **Отображение вашего любимого NFT**: Создайте свою собственную галерею, демонстрирующую ваши любимые NFT
- **Отображение аватара, сминтинного в игре**: Позвольте вашим пользователям создавать и затем просматривать сминтинный аватар

## Компоненты

Компонент `NFTCard` может быть кастомизирован с помощью следующих компонентов:

- `<NFTMedia square={boolean}/>` - Медиа для NFT, включает поддержку изображений, видео и аудио NFT.
- `<NFTTitle />` - Заголовок NFT.
- `<NFTOwner />` - Отображает аватар владельца, имя или адрес и значок (если применимо).
- `<NFTLastSoldPrice />` - Последняя цена продажи NFT в нативной валюте и USD.
- `<NFTNetwork />` - Сеть, в которой находится NFT, в настоящее время поддерживаются только NFT на BASE.

## Свойства

[`NFTCardProps`](#nftcardprops)

```ts
type NFTCardProps = {
  children?: React.ReactNode;
  /** Опциональное переопределение className для верхнего элемента div. */
  className?: string;
  /** Адрес контракта NFT */
  contractAddress: Hex;
  /** Обязательный идентификатор токена NFT */
  tokenId: string;
  /** Опциональный хук для переопределения хука useNftData по умолчанию */
  useNFTData?: UseNFTData;
  /** Опциональная функция обратного вызова для обработки ошибок внутри провайдера. */
  onError?: (error: NFTError) => void;
  /** Опциональная функция обратного вызова, которая предоставляет доступ к состоянию жизненного цикла компонента */
  onStatus?: (lifecycleStatus: LifecycleStatus) => void;
  /** Опциональная функция обратного вызова, которая предоставляет доступ к квитанции транзакции */
  onSuccess?: (transactionReceipt?: TransactionReceipt) => void;
};
```
