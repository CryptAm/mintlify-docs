Получайте лучшие результаты от AI-ассистентов для написания кода при работе с OnchainKit. Используете ли вы Cursor, GitHub Copilot или другие AI-инструменты - эти техники улучшат ваш рабочий процесс.

## Понимание контекстного окна

### Почему важен контекст

AI-ассистенты для написания кода имеют «контекстное окно» - объем текста, который они могут обработать за один раз. Это можно сравнить с оперативной памятью:

- Большинство ассистентов обрабатывают тысячи токенов (примерно 4-5 слов на токен)
- Все, что вы предоставляете, потребляет это ограниченное пространство
- Как только оно заполнено, более ранние части беседы могут быть утеряны

Предоставляйте релевантный контекст заранее, поскольку AI может работать только с тем, что находится в его текущем окне.

### Оптимизация под контекстные окна

Оптимизируйте работу с контекстными окнами:

1. **Предоставляйте релевантную информацию первой** - Самые важные детали в начале
2. **Пропускайте ненужный контент** - Избегайте нерелевантного кода или документации
3. **Структурируйте запросы четко** - Используйте разделы и форматирование
4. **Ссылайтесь разумно** - Показывайте только релевантные файлы для больших кодовых баз

Для крупных проектов создавайте центральный файл документации, а не объясняйте контекст повторно.

## Настройка AI-инструментов

### Правила Cursor (Cursor Rules)

Правила Cursor (Cursor Rules) обеспечивают постоянный контекст, помогая AI лучше понимать вашу кодовую базу.

1. Откройте палитру команд: `Cmd + Shift + P` (Mac) или `Ctrl + Shift + P` (Windows/Linux)
2. Найдите «Cursor Rules» и создайте/отредактируйте правила
3. Добавьте специфичные для проекта правила:
   - [Next.js](https://raw.githubusercontent.com/PatrickJS/awesome-cursorrules/refs/heads/main/rules/nextjs-tailwind-typescript-apps-cursorrules-prompt/.cursorrules)
   - [Astro](https://raw.githubusercontent.com/PatrickJS/awesome-cursorrules/refs/heads/main/rules/astro-typescript-cursorrules-prompt-file/.cursorrules)
   - [Vite](https://raw.githubusercontent.com/PatrickJS/awesome-cursorrules/refs/heads/main/rules/typescript-vite-tailwind-cursorrules-prompt-file/.cursorrules)
4. Сохраните, чтобы применить правила к AI-предложениям

### Проекты OnchainKit

Создайте новый проект OnchainKit:

```shell
npm create onchain@latest
```

После настройки сгенерируйте комплексную документацию для вашего проекта.

### Создание документации проекта

Файл с инструкциями проекта помогает AI понять вашу кодовую базу. Создайте его на раннем этапе и регулярно обновляйте.

**Промпт для Instructions.md:**

```
Создай детальный файл instructions.md для моего проекта OnchainKit:

1.  Обзор: Цели проекта и основная функциональность
2.  Технологический стек: Технологии, библиотеки, фреймворки с версиями
3.  Структура проекта: Организация файлов с объяснениями
4.  Стандарты кодирования: Стилевые соглашения, правила линтинга, паттерны
5.  Пользовательские сценарии: Ключевая функциональность с точки зрения пользователя
6.  API и интеграции: Внешние сервисы и подключения
```

<Note>
Для планирования архитектуры и сложных дизайнерских решений используйте AI-модели, ориентированные на рассуждения. Они отлично справляются с анализом компромиссов, крайних случаев и долгосрочным планированием.
</Note>


## Эффективные стратегии промптинга

### Будьте конкретны

AI-инструменты лучше всего реагируют на четкие и прямые инструкции.

❌ "Помоги с моим кодом"
✅ "Рефактори эту функцию аутентификации, чтобы использовать async/await вместо вложенных вызовов then()"

### Предоставляйте контекст

**Шаблон:**

```
Я работаю над проектом OnchainKit, используя [фреймворки/библиотеки]. Мне нужна помощь с:

1.  Проблема: [опишите конкретную задачу]
2.  Текущий подход: [что вы пробовали]
3.  Ограничения: [технические ограничения]
4.  Ожидаемый результат: [как выглядит успех]

Релевантная документация: @https://onchainkit.xyz

Релевантный код:
[вставьте ваш код]
```

### Итерируйте

Начните с простого и уточняйте, вместо того чтобы пытаться сделать все идеально с первого раза.

**Шаблон:**

```
Давай подойдем к этому шаг за шагом:
1.  Сначала реализуй базовый [функционал] с минимальной функциональностью
2.  Пересмотри и определи улучшения
3.  Добавь обработку ошибок и пограничные случаи
4.  Оптимизируй производительность

Начни с шага 1.
```

## Работа с OnchainKit

### Использование документации

Ссылайтесь на документацию OnchainKit прямо в своих промптах:

1. Просмотрите [OnchainKit docs](../getting-started/overview)
2. Найдите ваш компонент
3. Укажите URL документации в своем промпте

**Пример:**

```
Я реализую компонент обмена (swap) с OnchainKit.

Документация: @https://onchainkit.xyz

Покажи, как реализовать обмен, который:
1.  Обменивает USDC на ETH в сети Base
2.  Обрабатывает состояния подключения
3.  Включает обработку ошибок
4.  Следует лучшим практикам UX
```

### Интеграция компонентов

**Пример отображения баланса токена:**

```
Реализуй функцию отображения баланса токена:

1.  Показывает баланс подключенного кошелька для [TOKEN_SYMBOL]
2.  Обновляется при изменении баланса
3.  Обрабатывает состояния загрузки и ошибок
4.  Следует стандартам кодирования проекта
5.  Обнови instructions.md с этой реализацией
```

## Отладка с помощью AI

### Промпты для отладки

**Шаблон анализа ошибки:**

```
Я столкнулся с проблемой:

1.  Ожидаемое поведение: [что должно произойти]
2.  Фактическое поведение: [что происходит]
3.  Сообщения об ошибках: [если есть ошибки]
4.  Релевантный код: [проблемный код]

Проанализируй шаг за шагом и помоги мне:
1.  Определить возможные причины
2.  Предложить шаги для отладки
3.  Предложить решения
```

**Шаблон для добавления логирования отладки:**

```
Добавь комплексное логирование в эту функцию для отслеживания:
1.  Входных значений и их типов
2.  Потока выполнения функции
3.  Промежуточных изменений состояния
4.  Выходных значений или ошибок

Код:
[вставьте ваш код]
```

### Когда застряли

**Шаблон для уточнения:**

```
Я не уверен, как продолжить с [конкретной задачей]. Вот что я знаю:
1.  [контекст проблемы]
2.  [что вы пробовали]
3.  [конкретные области, где нужна помощь]

Какая дополнительная информация поможет тебе лучше помочь?
```

## Продвинутые техники

1. **Пошаговое рассуждение**
   ```
   Проанализируй этот код шаг за шагом и определи потенциальные проблемы.
   ```

2. **Спецификация формата**
   ```
   Структурируй свой ответ как обучающее руководство с примерами кода.
   ```

3. **Указание длины**
   ```
   Предоставь краткое объяснение в 2-3 абзаца.
   ```

4. **Уточнение неоднозначностей**
   ```
   Ты предложил два подхода. Я предпочитаю первый с TypeScript.
   ```

## Рекомендации

1. **Понимайте ограничения контекста** - AI имеет конечную память, расставляйте приоритеты для важной информации
2. **Предоставляйте релевантный контекст** - Показывайте код, ошибки и детали проекта, которые имеют значение
3. **Будьте конкретны** - Четкие инструкции лучше расплывчатых вопросов
4. **Разбивайте сложные задачи** - Итеративные подходы работают лучше
5. **Запрашивайте объяснения** - Просите AI объяснить сгенерированный код, который вы не понимаете
6. **Используйте четкое форматирование** - Структурируйте промпты разделами
7. **Ссылайтесь на документацию** - Включайте ссылки на документацию OnchainKit
8. **Тестируйте и проверяйте** - Всегда проверяйте код, сгенерированный AI, перед внедрением
9. **Накапливайте контекст** - Ссылайтесь на более ранние части разговора при итерациях
10. **Предоставляйте обратную связь** - Говорите AI, что сработало, а что нет
