---
title: Внесение вклада
---

Руководство по внесению вклада в разработку OnchainKit.

## Настройка

```bash
# Клонировать репозиторий
git clone git@github.com:coinbase/onchainkit.git

# Установить зависимости
pnpm install
```

Требуется Node.js и pnpm. См. [pnpm installation](https://pnpm.io/installation) если необходимо.

## Структура кодовой базы

Структура монорепозитория:

```bash
packages/onchainkit/
└── src/
   ├── api/                         - Компоненты и функции, связанные с API
   ├── core/                        - Файлы без зависимостей (zero dependencies)
   ├── styles/                      - Стили
   │   ├── index-with-tailwind.css  - Точка входа для CSS
   ├── {Component}/                 - Папка компонента
   │   ├── components/              - React-компоненты
   │   │   ├── {Name}.tsx
   │   │   ├── {Name}.test.tsx
   │   │   └── {Name}.css
   │   ├── core/                    - Утилити функции
   │   ├── index.ts                 - Точка входа для папки
   │   └── types.ts                 - Экспорт типов
   │
   ├── index.ts                     - Главная точка входа пакета
   ├── types.ts                     - Основные типы
   └── OnchainKitProvider.tsx       - Провайдер OnchainKit
```

## Разработка

```bash
# Запуск среды разработки с live reload
pnpm f:play dev

# Сборка пакета
pnpm f:ock build

# Запуск тестов
pnpm f:ock test
pnpm f:ock test:watch
pnpm f:ock test:coverage
```

Требования:
- 100% покрытие тестами для всех изменений
- Все тесты должны проходить
- Следование существующим паттернам кода

## Журнал изменений (Changelog)

Все изменения требуют записи в журнал изменений. Выполните эту команду и следуйте подсказкам:

```bash
pnpm changeset
```

Выберите подходящий тип обновления (обычно `minor`) и напишите четкое описание:

```
- **feat**: добавление render props в компонент ConnectWallet
- **fix**: исправление проблемы таймаута подключения кошелька
- **docs**: обновление примеров для компонента swap
```

Типы: `feat`, `fix`, `docs`, `chore`

Система автоматически добавляет ваше имя пользователя и номер PR при слиянии изменений.

## Что мы не принимаем

Мы не принимаем PR для:
- Опечаток в комментариях
- Изменений в тестах, которые не исправляют фактическую функциональность
- Правок в журнале изменений
- Незначительных изменений форматирования или стиля
