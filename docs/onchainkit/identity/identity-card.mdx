Компонент `IdentityCard` предоставляет комплексный способ отображения информации о пользовательской идентификации, включая имена ENS, аватары и разрешение имен, специфичных для цепей.

## Возможности
- **Разрешение имен**: Автоматически разрешает как Basenames, так и имена ENS
- **Поддержка аватаров**: Отображает аватары ENS и специфичные для цепей
- **Гибкий дисплей**: Настраиваемая компоновка и варианты стилей
- **Осведомленность о цепи**: Работает в разных EVM-цепях, поддерживающих разрешение имен
- **Поддержка подсказок**: Отображает информацию об аттестации при наведении на бейдж

## Настройка

Вы можете переопределить стили с помощью `className` или установив пользовательскую [тему OnchainKit](/onchainkit/guides/themes#custom-theme). Вы также можете установить цепь mainnet для разрешения имен ENS:

```tsx
import { IdentityCard } from '@coinbase/onchainkit/identity';
import { mainnet } from 'viem/chains'; // [!code focus]

<OnchainKitProvider
  config={{
    appearance: {
      mode: 'auto',
      theme: 'cyberpunk', // [!code focus]
    },
  }}
>
</OnchainKitProvider>

<IdentityCard 
  address="0x123..."
  chain={mainnet} // [!code focus]
  schemaId="0xf8b05c79f090979bf4a80270aba232dff11a10d9ca55c4f88de95317970f0de9"
/>
```

## Свойства (Props)

[`IdentityCardReact`](/onchainkit/identity/types#identitycardreact)

| Параметр | Тип  | Описание    |
|----------|------|-------------|
| `address` | `string` | Адрес кошелька, для которого отображается идентификация |
| `chain` | `Chain` | Цепь, на которой разрешается идентификация |
| `className` | `string` | Дополнительные CSS-классы для применения |
| `schemaId` | `Address \| null` | ID схемы для аттестации |
| `badgeTooltip` | `boolean \| string` | Когда `true` - отображает название аттестации в подсказке. Когда указана строка - показывает этот пользовательский текст. По умолчанию `false` |

## Обработка ошибок

Компонент корректно обрабатывает различные состояния ошибок:

- Невалидные адреса отображаются в сокращенном формате
- Отсутствующие имена ENS возвращаются к сокращенным адресам
- Неудачные загрузки аватаров показывают аватар по умолчанию
- Ошибки сети поддерживают деградированное, но функциональное отображение

## Рекомендации

1. Всегда предоставляйте валидный объект цепи из viem/chains
2. Обрабатывайте состояния загрузки в родительских компонентах, когда адрес может быть неопределен
3. Реализуйте правильные границы ошибок в родительских компонентах
4. Учитывайте адаптивность под мобильные устройства при стилизации
5. Используйте badgeTooltip для предоставления контекста о том, что представляет собой бейдж верификации
