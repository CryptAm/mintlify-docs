Важно понимать, в каком контексте используется ваш эндпоинт Paymaster, поскольку любой, у кого есть ваш ключ, может отправлять запросы на спонсирование транзакций.

Мы настоятельно рекомендуем настроить разрешенный список контрактов (allowlist) в конфигурации вашего paymaster. Это ограничит работу paymaster, позволяя ему спонсировать транзакции только для ваших конкретных контрактов.

Также рекомендуется применять дополнительные меры безопасности, такие как настройка прокси для paymaster, чтобы предотвратить утечку вашего API-ключа.

Если вы находитесь в ситуации, когда не можете добавить контракт в allowlist из-за динамических вызовов к контрактам, не указанным в политике, вы должны настроить прокси для paymaster на вашем бэкенде, чтобы ваш API-токен не был скомпрометирован.

Установите лимиты спонсирования, чтобы дополнительно гарантировать, что вы оплачиваете только то, что планируете.

## Прокси для Paymaster

Создание API для проксирования вызовов к вашему сервису paymaster важно по двум причинам:

- Позволяет защитить любой API-секрет.
- Позволяет добавить дополнительную валидацию для запросов, которые вы хотите спонсировать.

Вы можете найти больше деталей по реализации прокси для paymaster на сайте [smartwallet.dev](https://www.smartwallet.dev/guides/paymasters).
