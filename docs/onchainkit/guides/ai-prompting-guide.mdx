Это руководство помогает разработчикам эффективно использовать инструменты AI в своем рабочем процессе. Независимо от того, используете ли вы Cursor, GitHub Copilot или другие ИИ-ассистенты, эти стратегии помогут вам получать лучшие результаты и плавно интегрировать AI в процесс разработки.

## Понимание контекстных окон

### Почему контекст важен

AI-ассистенты для программирования имеют так называемое "контекстное окно" - объем текста, который они могут "видеть" и учитывать при генерации ответов. Представьте его как рабочую память AI:

- Большинство современных AI-ассистентов могут обрабатывать тысячи токенов (примерно 4-5 слов на токен)
- Все, что вы делитесь, и все, на что отвечает AI, потребляет это ограниченное пространство
- Как только контекстное окно заполняется, части истории вашего диалога могут быть утеряны.

Вот почему предоставление релевантного контекста заранее крайне важно - AI может работать только с тем, что он может "видеть" в своем текущем контекстном окне.

### Оптимизация для контекстных окон

Чтобы получить максимум от AI-ассистентов:

1. **Расставляйте приоритеты в информации**: В первую очередь делитесь самыми важными деталями.
2. **Удаляйте лишний контент:**: Избегайте вставки нерелевантного кода или документации.
3. **Структурируйте свои запросы**: Используйте четкие разделы и форматирование, чтобы информацию было легко обработать.
4. **Ссылайтесь на внешние ресурсы**: Для больших кодовых баз делитесь только самыми релевантными файлами.

Для крупных проектов создайте и используйте центральный файл документации, который суммирует ключевую информацию, вместо того чтобы повторно объяснять один и тот же контекст.

## Настройка AI-инструментов

### Настройка Cursor Rules

Cursor Rules позволяют предоставлять Cursor AI последовательный контекст, делая его более эффективным в понимании вашей кодовой базы и предоставлении релевантных предложений.

#### Создание Cursor Rules

1. Откройте палитру команд в Cursor:
   - Mac: `Cmd + Shift + P`
   - Windows/Linux: `Ctrl + Shift + P`
2. Найдите "Cursor Rules" и выберите опцию создания или редактирования правил

3. Добавьте правила, специфичные для проекта, которые помогут Cursor понять ваш проект:

   - [Next.js](https://raw.githubusercontent.com/PatrickJS/awesome-cursorrules/refs/heads/main/rules/nextjs-tailwind-typescript-apps-cursorrules-prompt/.cursorrules)
   - [Astro](https://raw.githubusercontent.com/PatrickJS/awesome-cursorrules/refs/heads/main/rules/astro-typescript-cursorrules-prompt-file/.cursorrules)
   - [Vite](https://raw.githubusercontent.com/PatrickJS/awesome-cursorrules/refs/heads/main/rules/typescript-vite-tailwind-cursorrules-prompt-file/.cursorrules)

4. Сохраните файл правил, и Cursor будет применять эти правила к своим ИИ-предложениям

### Настройка проекта OnchainKit

Чтобы создать новый проект OnchainKit:

```shell
npm create onchain@latest
```

После создания проекта составьте промпт для генерации комплексной документации для вашего нового проекта OnchainKit.

### Создание документации проекта

Комплексный файл с инструкциями помогает AI-инструментам лучше понять ваш проект. Его следует создать в начале проекта и регулярно обновлять.

**Готовый промпт для создания Instructions.md:**

```
Создай подробный файл instructions.md для моего проекта со следующими разделами:

1. Обзор: Кратко опиши цели проекта, проблемные утверждения и основную функциональность.
2. Технологический стек: Перечисли все технологии, библиотеки, фреймворки с версиями.
3. Структура проекта: Опиши организацию файлов с пояснениями.
4. Стандарты кодирования: Документируй стилистические соглашения, правила линтинга и паттерны.
5. Пользовательские сценарии: Ключевая функциональность с точки зрения пользователя.
6. API и интеграции: Внешние сервисы и то, как они подключаются.
```

<Note>
Примечание: При планировании архитектуры или принятии сложных проектных решений используйте ИИ-модели с развитым логическим мышлением, такие как o4 mini или Claude 3.7 Sonnet. Они отлично справляются с анализом компромиссов, граничных случаев и долгосрочным планированием.
</Note>


## Эффективные стратегии составления промптов

### Будьте конкретными и прямыми

Начинайте с четких команд и будьте конкретны в том, что хотите. ИИ-инструменты лучше всего реагируют на ясные, прямые инструкции.

**Пример:** ❌ "Помоги с моим кодом"  
 ✅ "Рефактори эту функцию аутентификации, чтобы использовать async/await вместо вложенных вызовов then()"

### Предоставляйте контекст для сложных задач

**Готовый промпт:**

```
 Я работаю над проектом на onchainkit, использующим [фреймворки/библиотеки]. Мне нужна твоя помощь с:

1. Проблема: [опиши конкретную проблему]
2. Текущий подход: [объясни, что ты пробовал]
3. Ограничения: [упомяни любые технические ограничения]
4. Ожидаемый результат: [опиши, как выглядит успешное решение]

Вот релевантная документация @https://docs.base.org/onchainkit/getting-started

Вот релевантный код:
[вставь свой код]
```

### Просите об итерациях

Начинайте с простого и уточняйте через итерации, вместо того чтобы пытаться получить все идеально с первого раза.

**Готовый промпт:**

```
Давай подойдем к этому шаг за шагом:
1. Сначала реализуем базовую версию [фичи] с минимальной функциональностью.
2. Затем мы просмотрим ее и определим области для улучшения.
3. Далее добавим обработку ошибок и граничные случаи.
4. Наконец, мы оптимизируем производительность.

Пожалуйста, начни с шага 1 сейчас.
```

## Работа с OnchainKit

### Использование LLMs.txt для документации

Проект OnchainKit предоставляет оптимизированную документацию в виде файлов LLMs.txt. Эти файлы специально отформатированы для использования ИИ-моделями:

1. Используйте [документацию OnchainKit](/onchainkit/getting-started)
2. Найдите компонент, который хотите реализовать
3. Скопируйте соответствующий URL LLMs.txt
4. Вставьте его в свой промпт, чтобы предоставить контекст

**Пример использования LLMs.txt:**

```
Я реализую компонент обмена (swap) с помощью OnchainKit. Вот релевантный LLMs.txt:

@https://docs.base.org/onchainkit/getting-started

Основываясь на этой документации, пожалуйста, покажи мне, как реализовать коннектор кошелька, который:
1. Осуществляет обмен (swap) с Base USDC на Base ETH.
2. Правильно обрабатывает состояния подключения.
3. Включает обработку ошибок.
4. Следует лучшим практикам пользовательского опыта.
```

### Пример интеграции компонента

**Готовый промпт для отображения баланса токенов:**

```
Мне нужно реализовать новую функциональность в моем проекте, которая:

1. Показывает баланс {ERC20-токена} подключенного кошелька.
2. Обновляется при изменении баланса.
3. Правильно обрабатывает состояния загрузки и ошибок.
4. Следует стандартам кодирования нашего проекта.
5. Обновляет instructions.md, отражая эту новую реализацию.

```

## Отладка с помощью AI

### Эффективные промпты для отладки

**Готовый промпт для анализа бага:**

```
Я столкнулся с проблемой в моем коде:

1. Ожидаемое поведение: [что должно происходить]
2. Фактическое поведение: [что происходит на самом деле]
3. Сообщения об ошибках: [включи любые ошибки]
4. Релевантный код: [вставь проблемный код]

Пожалуйста, проанализируй эту ситуацию шаг за шагом и помоги мне:
1. Определить потенциальные причины этой проблемы
2. Предложить шаги отладки для изоляции проблемы
3. Предложить возможные решения
```

**Готовый промпт для добавления логов отладки:**

```
Мне нужно отладить следующую функцию. Пожалуйста, добавь комплексные логирующие операторы, которые помогут мне отследить:
1. Входные значения и их типы
2. Поток выполнения функции
3. Промежуточные изменения состояния
4. Выходные значения или ошибки

Вот мой код:
[вставь свой код]
```

### Когда вы застряли

Если вы не уверены, как действовать дальше:

**Готовый промпт для уточнения:**

```
Я не уверен, как продолжить с [конкретной задачей]. Вот что я знаю:
1. [контекст о проблеме]
2. [что ты пробовал]
3. [конкретные области, где тебе нужно руководство]

Какая дополнительная информация поможет тебе оказать лучшую помощь?
```

## Продвинутые техники составления промптов

Современные ИИ-ассистенты обладают возможностями, которые вы можете использовать с помощью этих продвинутых техник:

1. **Пошаговое рассуждение**: Просите ИИ решать проблемы систематически

```
Пожалуйста, проанализируй этот код шаг за шагом и определи потенциальные проблемы.
```

2.  **Спецификация формата**: Запрашивайте конкретные форматы для ясности

```
Пожалуйста, структурируй свой ответ как учебное пособие с примерами кода и пояснениями.
```

3.  **Указание объема**: Указывайте, хотите ли вы краткий или подробный ответ

```
Пожалуйста, предоставь краткое объяснение в 2-3 абзаца.
```

4.  **Прояснение неоднозначностей**: Помогайте разрешать неясные моменты, когда получаете несколько вариантов

```
Я заметил, ты предложил два подхода. Чтобы прояснить, я бы предпочел использовать первый подход с TypeScript.
```

## Краткое изложение лучших практик

1. **Понимайте ограничения контекста**: Осознавайте, что у ИИ-инструментов ограниченные контекстные окна, и расставляйте приоритеты в информации соответствующим образом.
2. **Предоставляйте релевантный контекст**: Делитесь фрагментами кода, сообщениями об ошибках и деталями проекта, которые имеют значение для вашего конкретного вопроса.
3. **Будьте конкретны в запросах**: Четкие, прямые инструкции дают лучшие результаты, чем расплывчатые вопросы.
4. **Разбивайте сложные задачи на шаги**: Итеративные подходы часто работают лучше для сложных проблем.
5. **Просите объяснений**: Просите AI объяснить сгенерированный код или понятия, которые вы не понимаете.
6. **Используйте форматирование для ясности**: Структурируйте свои промпты четкими разделами и форматированием.
7. **Ссылайтесь на документацию**: При работе с конкретными библиотеками, такими как OnchainKit, делитесь соответствующей документацией.
8. **Тестируйте и проверяйте**: Всегда проверяйте и тестируйте код, сгенерированный ИИ, перед внедрением.
9. **Стройте на основе предыдущего контекста**: Ссылайтесь на более ранние части вашего диалога при итерациях.
10. **Давайте обратную связь**: Сообщайте ИИ, что сработало, а что нет, чтобы улучшить будущие ответы.
