## Настройка

### Клонируйте репозиторий

```bash
git clone git@github.com:coinbase/onchainkit.git
```

### Установите Node и pnpm

Используйте nvm, mise, n или ваш любимый менеджер версий для установки Node.js.

Для pnpm ознакомьтесь с инструкциями по установке на [сайте pnpm](https://pnpm.io/installation).

### Установите зависимости

Из корня репозитория:

```bash
pnpm install
```

## Codebase

Этот проект является монорепозиторием, управляемым с помощью pnpm. Пакет `@coinbase/onchainkit` находится в:

```bash
packages/onchainkit/
```

Вот примерная структура codebase:

```bash
packages/onchainkit/
└── src/
   ├── api/                         - Компоненты и функции, связанные с API
   ├── core/                        - Файлы без зависимостей
   ├── styles/                      - Стили
   │   ├── index-with-tailwind.css  - Точка входа для CSS
   ├── {Component}/                 - Папка компонента
   │   ├── components/              - React-компоненты
   │   │   ├── {Name}.tsx
   │   │   ├── {Name}.test.tsx
   │   │   └── {Name}.css
   │   ├── core/                    - Утилити
   │   ├── index.ts                 - Точка входа для папки
   │   └── types.ts                 - Экспорт типов
   │
   ├── index.ts                     - Главная точка входа пакета
   ├── types.ts                     - Основные типы
   └── OnchainKitProvider.tsx       - Провайдер OnchainKit
```

## Рабочие процессы

### Разработка

Для работы над компонентами OnchainKit с живой обратной связью в интерфейсе:

```bash
pnpm f:play dev
```

Это соберет пакет OnchainKit в режиме отслеживания изменений (watch mode) и запустит среду разработки (площадку для экспериментов), где вы сможете видеть свои компоненты в действии.

По мере внесения изменений площадка будет обновляться автоматически.

Перейдите по адресу [http://localhost:3000](http://localhost:3000),чтобы открыть площадку для экспериментов.

### Сборка

Для сборки пакета:

```bash
pnpm f:ock build
```

### Тестирование

Пишите и обновляйте существующие модульные тесты. Вы можете запустить тесты с помощью:

```bash
pnpm f:ock test
```

Для отслеживания изменений файлов и автоматического перезапуска тестов:

```bash
pnpm f:ock test:watch
```

Мы ожидаем 100% покрытия кода для любых обновлений. Вы можете получить информацию о покрытии с помощью:

```bash
pnpm f:ock test:coverage
```

Если покрытие падает ниже 100%, просмотрите отчет о покрытии, сгенерированный вышеуказанной командой:

```bash
open coverage/index.html
```

### Обновление журнала изменений (changelog)

Для обновления журнала изменений выполните:

```bash
pnpm changeset
```

Выберите `minor` и используйте следующий формат для описания:

```markdown
- **feat**: Информация об обновлении функции. By @your-github-id #XX (XX - номер PR)
```

Возможные значения:

- `feat`
- `fix`
- `docs`
- `chore`

## Запрос на новую функцию

Есть идея для компонента, который мы еще не поддерживаем? Вы можете отправить запрос на новую функцию в наш [Github](https://github.com/coinbase/onchainkit/issues). Создайте "New issue" и пометьте его как "Feature Request: ...".
