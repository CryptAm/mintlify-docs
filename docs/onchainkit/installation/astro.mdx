Установите и настройте OnchainKit с Astro.
Если вы интегрируете OnchainKit в существующий проект,
перейдите к разделу [установки OnchainKit](#install-onchainkit).

<Steps>
  <Step title="Установите Astro">
  Создайте новый проект Astro, используя CLI Astro.
  Подробнее об Astro можно узнать [здесь](https://docs.astro.build/en/install-and-setup/#start-a-new-project).

  <CodeGroup>
  ```bash npm
  npm create astro@latest
  ```

  ```bash yarn
  yarn create astro
  ```

  ```bash pnpm
  pnpm create astro@latest
  ```
  </CodeGroup>
  </Step>
  <Step title="Установите React">
  Astro по умолчанию не включает React, поэтому, если вы еще не используете React в своем приложении, 
  вам нужно будет его установить.

  ```bash Terminal
  npx astro add react
  ```
  </Step>
  <Step title="Установите OnchainKit">
  Добавьте OnchainKit в свой проект, установив пакет `@coinbase/onchainkit`.

  <CodeGroup>
  ```bash npm
  npm install @coinbase/onchainkit
  ```

  ```bash yarn
  yarn add @coinbase/onchainkit
  ```

  ```bash pnpm
  pnpm add @coinbase/onchainkit
  ```

  ```bash bun
  bun add @coinbase/onchainkit
  ```
  </CodeGroup>
  </Step>
  <Step title="Получите Client API Key">
  Получите свой [Client API Key](https://portal.cdp.coinbase.com/projects/api-keys/client-key) от Coinbase Developer Platform.

<Frame>
  <img
    alt="Скопируйте Client API Key для OnchainKit"
    src="/images/onchainkit/copy-api-key-guide.png"
    height="364"
  />
</Frame>

  Создайте файл `.env` в корневом каталоге вашего проекта.

<Frame>
  <img
    alt="OnchainKit define Client API Key"
    src="/images/onchainkit/getting-started-create-env-file.png"
    width="250"
    loading="lazy"
  />
</Frame>

  Добавьте свой Client API Key в файл `.env`:

  ```dotenv .env
  PUBLIC_ONCHAINKIT_API_KEY=YOUR_CLIENT_API_KEY
  ```
  </Step>
  <Step title="Добавьте провайдеры">
  Создайте файл `providers.tsx`. Добавьте `OnchainKitProvider` с нужной вам конфигурацией.

  Под капотом OnchainKit создаст наши рекомендованные провайдеры Wagmi и QueryClient. 
  Если вы хотите настроить эти провайдеры, ознакомьтесь с разделом [Пользовательские 
  дополнительные провайдеры](/onchainkit/config/supplemental-providers).

  ```tsx providers.tsx
  // @noErrors: 2307 2580 2339 - cannot find 'process', cannot find './wagmi', cannot find 'import.meta'
  'use client';

  import type { ReactNode } from 'react';
  import { OnchainKitProvider } from '@coinbase/onchainkit';
  import { base } from 'wagmi/chains'; // добавьте baseSepolia для тестирования // [!code ++]

  export function Providers(props: { children: ReactNode }) {
    return (
      <OnchainKitProvider
        apiKey={import.meta.env.PUBLIC_ONCHAINKIT_API_KEY} // [!code ++]
        chain={base} // добавьте baseSepolia для тестирования // [!code ++]
      >
        {props.children}
      </OnchainKitProvider>
    );
  }
  ```
  </Step>
  <Step title="Оберните ваши компоненты OnchainKit в <AppProviders />">
  После настройки провайдеров в шаге 4 вам нужно будет обернуть ваши компоненты OnchainKit в 
  компонент `<AppProviders />`.

  Есть два варианта для этого:

  1. Создайте компонент, например `<ReactIsland />`, который содержит все компоненты OnchainKit.
  2. Оберните каждый компонент OnchainKit индивидуально.

  <CodeGroup>
  ```tsx ReactIsland
  import { AppProviders } from '../AppProviders';

  export default function ReactIsland() {
    return (
      <AppProviders>
        <YourReactAppContainingOnchainKitComponents />
      </AppProviders>
    );
  }
  ```

  ```tsx OnchainKit Wrappers
  import { AppProviders } from '../AppProviders';
  import { OnchainKitComponent } from '@coinbase/onchainkit';

  export default function OnchainKitComponentWrapper() {
    return (
      <AppProviders>
        <OnchainKitComponent />
      </AppProviders>
    );
  }
  ```
  </CodeGroup>

  Преимущество ReactIsland в том, что у вас будет только один провайдер в любой момент времени.
  Недостаток в том, что все ваши компоненты OnchainKit должны находиться в одном Island.

  Преимущество индивидуальных оберток в том, что вы можете использовать компоненты OnchainKit в любом месте вашего приложения.
  Недостаток в том, что у вас будет несколько провайдеров, если вы используете более одного компонента OnchainKit.
  </Step>
  <Step title="Добавьте компоненты OnchainKit в ваше приложение">
  Вы можете добавить компоненты OnchainKit в ваше приложение, используя компонент(ы), которые вы
  создали выше, в ваши `.astro` файлы.

  Например, если вы создали ReactIsland, вы можете добавить его в ваш
  файл `src/pages/index.astro`:

  ```astro src/pages/index.astro
  ---
  import Layout from '../layouts/Layout.astro';
  import ReactIsland from '../components/ReactIsland';
  ---

  <Layout title="Welcome to Astro.">
    <main>
      ...
      <ReactIsland client:only="react" />
      ...
    </main>
  </Layout>
  ```

  Не забудьте добавить директиву `client:only="react"` к вашему компоненту OnchainKit,
  так как это требуется для рендеринга React-компонентов в Astro.
  </Step>
  <Step title="Импортируйте стили OnchainKit">
    Компоненты OnchainKit поставляются с предварительно настроенными стилями.
  Чтобы включить эти стили в ваш проект, добавьте следующий импорт
  в начало файла `Layout.astro`:

  ```tsx
  import '@coinbase/onchainkit/styles.css';
  ```

  Это гарантирует, что стили OnchainKit будут загружены и применены ко всему вашему приложению.

  - Для пользователей Tailwind CSS ознакомьтесь с нашим [руководством по интеграции Tailwind](/onchainkit/guides/tailwind).

  - Обновите внешний вид компонентов, используя наши встроенные темы или создав свою собственную пользовательскую тему.
    Изучите возможности в нашем [Руководстве по темам](/onchainkit/guides/themes).
  </Step>
</Steps>
