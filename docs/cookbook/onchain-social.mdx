## Что такое ончейн социальные приложения?

Традиционные социальные сети следуют знакомой, но ограничивающей схеме: пользователи входят с адресами электронной почты, прокручивают централизованные ленты и кликают по ссылкам, которые перенаправляют их в сторону от беседы. Хотя эта модель работает, у нее есть фундаментальные ограничения, которые ончейн социальные приложения стремятся решить.

### Парадигма ончейн социальных приложений

Ончейн социальные приложения представляют собой фундаментальный сдвиг в том, как мы думаем о цифровой идентичности и социальных взаимодействиях:

**Проблемы традиционной модели:**
- Ваша идентичность, контент и взаимодействия принадлежат платформе
- Разработка ведется поверх платформ, никогда внутри них
- Пользователям приходится покидать беседы для доступа к новому опыту
- Разработчики ограничены API и правилами платформ

**Решения ончейн социальных приложений:**
- **Переносимая идентичность**: Ваш Farcaster ID (FID) принадлежит вам, а не привязан к какой-либо одной платформе
- **Встроенные возможности**: Богатые, интерактивные приложения работают нативно внутри постов и бесед
- **Свобода разработчика**: Создавайте как полноценные участники социального графа
- **Владение пользователя**: Контролируйте свои данные, контент и социальные связи

Речь не идет о замене существующих платформ - речь идет о снятии стен между пользователями, разработчиками и опытом, который они создают вместе.
</Info>

## Мини-приложения: Интерфейс для ончейн социальных приложений

Мини-приложения - это легковесные, выразительные веб-приложения, которые живут прямо внутри социальных лент. Они запускаются мгновенно без установки и предоставляют богатый, интерактивный опыт, реагирующий на вашу ончейн идентичность.

**Что делает мини-приложения особенными:**
- **Встроенный опыт**, не перенаправляющий пользователей в сторону от бесед
- **Осведомленность о социальном контексте* - они знают, кто их открыл и откуда
- **Интеграция с ончейн идентичностью** с автоматической персонализацией
- **Нативный вид и ощущения** внутри социальных платформ

**Примеры использования:**
- Интерактивные опросы и голосования в реальном времени
- Социальные игры и развлечения
- Электронная коммерция с мгновенной оплатой
- Интерфейсы DeFi и управление портфелем
- Демонстрация и торговля NFT
- Инструменты для совместного принятия решений

<Tip>
Мини-приложения предоставляют разработчикам прямой доступ к социальной дистрибуции - вы строите внутри беседы, а не пытаетесь отвлечь пользователей от нее.
</Tip>

## Создайте свое первое мини-приложение

<Steps>
  <Step title="Установите MiniKit и создайте проект">
    Самый быстрый способ создать мини-приложение - использовать MiniKit, который автоматически обрабатывает аутентификацию, социальный контекст и ончейн интеграции.

    **Создайте новое мини-приложение:**
    ```bash
    npx create-onchain --mini
    cd your-mini-app-name
    npm install
    ```

    **Запустите сервер для разработки:**
    ```bash
    npm run dev
    ```

    <Check>
    Ваше мини-приложение теперь должно работать по адресу `http://localhost:3000` с полностью функциональным социальным интерфейсом.
    </Check>

    **Что вы получаете из коробки:**
    - Полный каркас фронтенда и бэкенда
    - Встроенная поддержка идентичности Farcaster и уведомлений
    - Нативная интеграция с блокчейном Base и OnchainKit
    - Адаптивный дизайн, оптимизированный для мобильных социальных лент
    - Инструменты разработки для тестирования и отладки

    <Tip>
    Каркас MiniKit включает примеры компонентов и страниц, которые помогут вам быстро понять архитектуру.
    </Tip>
  </Step>

  <Step title="Поймите социальный контекст">
    Мини-приложения имеют доступ к богатому социальному контексту, которого лишены традиционные веб-приложения. Этот контекст делает персонализацию легкой и позволяет создавать   интеллектуальные взаимодействия.

    **Получите доступ к контексту пользователя и соцсети:**
    ```tsx
    import { useMiniKit } from "@coinbase/onchainkit/minikit";

    export default function App() {
      const { context } = useMiniKit();

      // Проверенная информация о пользователе (всегда доступна)
      const userFid = context?.user?.fid;
      
      // Дополнительные данные пользователя (когда доступны)
      const username = context?.user?.username;
      const displayName = context?.user?.displayName;
      const pfpUrl = context?.user?.pfpUrl;

      // Информация о клиенте
      const isAdded = context?.client?.added; // Добавил ли пользователь это мини-приложение
      const location = context?.location; // Откуда было запущено мини-приложение

      return (
        <div className="mini-app">
          <h1>Welcome{displayName ? `, ${displayName}` : ''}!</h1>
          <p>Your FID: {userFid}</p>
          {username && <p>Username: @{username}</p>}
          {isAdded && <p>✅ You've added this Mini App</p>}
          {location && <p>Launched from: {location}</p>}
        </div>
      );
    }
    ```

    **Проверенные свойства контекста:**
    - **Идентификатор пользователя**: FID (всегда доступен), имя пользователя, отображаемое имя, URL изображения профиля
    - **Статус клиента**: Добавил ли пользователь ваше мини-приложение в избранное
    - **Контекст запуска**: Общая информация о том, откуда было открыто мини-приложение
    - **Отступы безопасной зоны**: Для корректной мобильной верстки внутри клиентов Farcaster

    <Warning>
    Всегда обрабатывайте случаи, когда контекст может быть неопределен, особенно во время разработки и тестирования.
    </Warning>
  </Step>

  <Step title="Создайте интерактивные функции">
    Создавайте увлекательный социальный опыт, использующий встроенную природу мини-приложения:

    **Пример: Приложение для социального голосования**
    ```tsx
    import { useState, useEffect } from 'react';
    import { useMiniKit } from "@coinbase/onchainkit/minikit";

    export default function VotingApp() {
      const { context } = useMiniKit();
      const [votes, setVotes] = useState({ option1: 0, option2: 0 });
      const [userVote, setUserVote] = useState(null);

      const handleVote = async (option) => {
        if (userVote) return; // Предотвращаем повторное голосование

        // Отправляем голос на ваш бэкенд
        const response = await fetch('/api/vote', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            fid: context?.user?.fid,
            option,
            location: context?.location
          })
        });

        if (response.ok) {
          setUserVote(option);
          // Update vote counts
          setVotes(prev => ({
            ...prev,
            [option]: prev[option] + 1
          }));
        }
      };

      return (
        <div className="voting-app">
          <h2>What's your preference?</h2>
          
          <div className="vote-options">
            <button 
              onClick={() => handleVote('option1')}
              disabled={userVote}
              className={userVote === 'option1' ? 'selected' : ''}
            >
              Option A ({votes.option1} votes)
            </button>
            
            <button 
              onClick={() => handleVote('option2')}
              disabled={userVote}
              className={userVote === 'option2' ? 'selected' : ''}
            >
              Option B ({votes.option2} votes)
            </button>
          </div>

          {userVote && (
            <p>Thanks for voting! Your choice: {userVote}</p>
          )}
        </div>
      );
    }
    ```

    <Tip>
    Используйте FID пользователя, чтобы предотвратить дублирование голосов и создавать персонализированный опыт без отдельной аутентификации.
    </Tip>
  </Step>

  <Step title="Добавьте интеграцию с блокчейном">
    Усильте ваше мини-приложение блокчейн-функциональностью с помощью OnchainKit:

    ```tsx
    import { 
      Transaction, 
      TransactionButton, 
      TransactionStatus 
    } from '@coinbase/onchainkit/transaction';

    export default function OnchainMiniApp() {
      const { context } = useMiniKit();
      
      const handleTransaction = async () => {
        // Создаем транзакцию на основе социального контекста
        const txData = {
          to: '0x...',
          value: '1000000000000000000', // 1 ETH
          data: '0x...'
        };
        
        return txData;
      };

      return (
        <div className="onchain-mini-app">
          <h2>Onchain Social Action</h2>
          <p>Initiated by FID: {context?.user?.fid}</p>
          
          <Transaction
            calls={[handleTransaction]}
            onSuccess={(receipt) => {
              console.log('Transaction successful:', receipt);
            }}
          >
            <TransactionButton text="Execute Onchain Action" />
            <TransactionStatus />
          </Transaction>
        </div>
      );
    }
    ```

    <Check>
    Ваше мини-приложение теперь может инициировать блокчейн-транзакции прямо из социальных взаимодействий.
    </Check>
  </Step>

  <Step title="Задеплойте и протестируйте ваше мини-приложение">
    Задеплойте ваше мини-приложение, чтобы сделать его доступным внутри социальных лент:

    **Деплой в Vercel (рекомендуется):**
    ```bash
    npm install -g vercel
    vercel --prod
    ```

    **Протестируйте ваше мини-приложение:**
    1. **Локальное тестирование**: Используйте инструменты разработки MiniKit
    2. **Тестирование как Frame**: Тестируйте как Farcaster Frame
    3. **Социальное тестирование**: Деплойте и тестируйте в реальных социальных контекстах
    4. **Тестирование производительности**: Убедитесь в быстрой загрузке в мобильных окружениях

    <Warning>
    Всегда тестируйте ваше мини-приложение в реальных социальных контекстах перед широким деплоем, так как социальная среда может влиять на производительность и пользовательский опыт.
    </Warning>
  </Step>
</Steps>

- Настройте переменные окружения и деплой
