Прежде чем делиться вашим мини-приложением, убедитесь, что все работает.

## Чек-лист перед деплоем

- Приложение задеплоено на публичном HTTPS домене
- Переменные окружения установлены на вашей платформе для деплоя
- `/.well-known/farcaster.json` возвращает валидный JSON
- Метаданные `fc:frame` при публикации отображают кнопку запуска

## Инструменты для валидации

[Base Build](https://base.dev) это место для тестирования и валидации вашего мини-приложения. Помимо отслеживания использования вашего приложения и получения наград для разработчиков, он включает **инструмент предварительного просмотра (Preview tool)**, который помогает вам:

- Проверить манифест и метаданные вашего приложения
- Протестировать, как ваше приложение будет отображаться в Base app
- Подтвердить владение и ассоциацию с аккаунтом

Инструмент предварительного просмотра предоставляет четкие визуальные подсказки:
- ✅ Зеленые галочки, когда все настроено правильно
- ❌ Красные индикаторы, когда что-то требует вашего внимания

### Components of the Preview Tool  

Компоненты инструмента предварительного просмотра

- **Консоль**: Preview your app and review logs to make informed decisions about performance.  
- **Ассоциация с аккаунтом (Account Association)**: Подтвердите, что ваше приложение привязано к правильному аккаунту, подписи валидны, и домен соответствует указанному в манифесте.
- **Метаданные**: Убедитесь, что ваше мини-приложение отображается именно так, как ожидается, проверив обязательные поля, такие как имя, иконка, теги и изображение заставки (splash image).

### Начало работы

1. [Импортируйте ваше приложение](https://docs.base.org/mini-apps/growth/data-driven-growth) в Base Build
2. Посетите [инструмент предварительного просмотра](https://base.dev/preview) чтобы начать валидацию вашего приложения



## Поделитесь и протестируйте

- Создайте каст с URL вашего приложения
- Проверьте превью и кнопку запуска
- Протестируйте запуск и готовность фрейма
