<Info>
**Для большинства пользователей:** Используйте существующие платформы для запуска токенов, такие как Zora, Clanker или Flaunch. Эти инструменты берут на себя техническую сложность, предоставляя уникальные функции для различных случаев использования.

**Для разработчиков:** Создавайте пользовательские ERC-20 токены с помощью Foundry и протестированных контрактов OpenZeppelin для максимального контроля и кастомизации.
</Info>

## Выбор подхода к запуску

### Запуск через платформу (рекомендуется для большинства пользователей)

Выберите платформу, когда хотите:
- Быстрое развертывание без написания кода
- Встроенные функции для сообщества
- Автоматическое управление ликвидностью
- Возможности социальной интеграции

### Пользовательская разработка (для разработчиков)

Создайте свой собственный смарт-контракт, когда вам нужно:
- Пользовательская токеномика или функциональность
- Полный контроль над поведением контракта
- Интеграция с существующими системами
- Расширенные требования безопасности

## Платформы для запуска токенов на Base

### Zora
**Лучше всего подходит для:** Создателей контента и социальных токенов

Zora превращает каждый пост в торгуемый ERC-20 токен с автоматической интеграцией Uniswap. Каждый пост становится "монетой" с эмиссией 1 миллиард единиц, создатели получают 10 миллионов токенов и зарабатывают 1% от всех торговых комиссий.

**Ключевые особенности:**
- Создание токенов, ориентированных на соцсети
- Автоматические пулы ликвидности
- Разделение дохода для создателей
- Встроенный интерфейс для торговли

[Начать работу с Zora →](https://zora.co)

### Clanker
**Лучше всего подходит для:** Быстрого запуска мемкоинов через социальные сети

Clanker - это инструмент для развертывания токенов с использованием AI, работающий через Farcaster. Пользователи могут создавать ERC-20 токены на Base, просто отмечая @clanker со своей концепцией токена.

**Ключевые особенности:**
- Автоматизация на основе AI
- Интеграция с социальными сетями через Farcaster
- Мгновенное развертывание
- Обнаружение, управляемое сообществом

[Начать работу с Clanker →](https://farcaster.xyz) или посетите [clanker.world](https://clanker.world)

### Flaunch
**Best for:** Advanced memecoin projects with sophisticated tokenomics

Flaunch использует Uniswap V4 для реализации программируемого распределения доходов, автоматических выкупов (buybacks) и технологии Progressive Bid Walls для поддержки цены. Создатели могут настраивать распределение комиссий и управление казной.

**Ключевые особенности:**
- Программируемое разделение доходов
- Механизмы автоматического выкупа
- Технология Progressive Bid Wall
- Инструменты управления казной

[Начать работу с Flaunch →](https://flaunch.gg)

### Mint Club
**Лучше всего подходит для:** Семейств токенов и активов, управляемых сообществом

Mint Club позволяет любому запустить новые токены или NFT, обеспеченные существующими ERC-20 токенами. Каждая сделка блокирует родительский токен в пуле кривой связывания (bonding curve), создавая управляемую автоматически ончейн ликвидность, которая усиливает родительский токен и поддерживает новые активы сообщества.

**Ключевые особенности:**
- Запуск токенов или NFT из любого ERC-20 токена
- Автоматизированная ликвидность с настраиваемыми кривыми связывания
- Интегрированный торговый интерфейс с кросс-чейн свопом
- Встроенные инструменты для airdrop и блокировки
- Простая торговля токенами и NFT
- Разделение дохода для создателей

[Начать работу с Mint Club →](https://mint.club)

## Техническая реализация с Foundry

Для разработчиков, которые хотят полного контроля над реализацией своего токена, вот как создать и развернуть пользовательский ERC-20 токен на Base с использованием Foundry.

<Warning>
Перед запуском пользовательского токена в продакшен всегда проводите аудит безопасности с привлечением экспертов по смарт-контрактам.
</Warning>

### Настройка проекта

Инициализируйте новый проект Foundry и очистите файлы шаблона:

```bash Terminal
# Создаем новый проект
forge init my-token-project
cd my-token-project

# Удаляем ненужные файлы шаблона
rm src/Counter.sol script/Counter.s.sol test/Counter.t.sol
```

Установите контракты OpenZeppelin для безопасной, проверенной реализации ERC-20:

```bash Terminal
# Устанавливаем библиотеку контрактов OpenZeppelin
forge install OpenZeppelin/openzeppelin-contracts
```

### Разработка смарт-контракта

Создайте контракт токена, используя реализацию ERC-20 от OpenZeppelin:

```solidity src/MyToken.sol
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.19;

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
import "@openzeppelin/contracts/access/Ownable.sol";

/**
 * @title MyToken
 * @dev ERC-20 токен с возможностью минта и ограничением эмиссии
 */
contract MyToken is ERC20, Ownable {
    // Максимальное количество токенов, которое может существовать
    uint256 public constant MAX_SUPPLY = 1_000_000_000 * 10**18; // 1 миллиард токенов
    
    constructor(
        string memory name,
        string memory symbol,
        uint256 initialSupply,
        address initialOwner
    ) ERC20(name, symbol) Ownable(initialOwner) {
        require(initialSupply <= MAX_SUPPLY, "Initial supply exceeds max supply");
        // Минтим начальную эмиссию развертывающему контракт
        _mint(initialOwner, initialSupply);
    }
    
    /**
     * @dev Минт новых токенов (только владелец контракта может вызывать)
     * @param to Адрес для получения новых токенов
     * @param amount Количество токенов для минта
     */
    function mint(address to, uint256 amount) public onlyOwner {
        require(totalSupply() + amount <= MAX_SUPPLY, "Minting would exceed max supply");
        _mint(to, amount);
    }
    
    /**
     * @dev Сжигание токенов с баланса вызывающего
     * @param amount Количество токенов для сжигания
     */
    function burn(uint256 amount) public {
        _burn(msg.sender, amount);
    }
}
```

### Скрипт развертывания

Создайте скрипт развертывания в соответствии с рекомендациями Foundry:

```solidity script/DeployToken.s.sol
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.19;

import {Script, console} from "forge-std/Script.sol";
import {MyToken} from "../src/MyToken.sol";

contract DeployToken is Script {
    function run() external {
        // Загружаем приватный ключ развертывающего из переменных окружения
        uint256 deployerPrivateKey = vm.envUint("PRIVATE_KEY");
        address deployerAddress = vm.addr(deployerPrivateKey);
        
        // Параметры конфигурации токена
        string memory name = "My Token";
        string memory symbol = "MTK";
        uint256 initialSupply = 100_000_000 * 10**18; // 100 миллионов токенов
        
        // Начинаем трансляцию транзакций
        vm.startBroadcast(deployerPrivateKey);
        
        // Развертываем контракт токена
        MyToken token = new MyToken(
            name,
            symbol,
            initialSupply,
            deployerAddress
        );
        
        // Заканчиваем трансляцию транзакций
        vm.stopBroadcast();
        
        // Логируем информацию о развертывании
        console.log("Token deployed to:", address(token));
        console.log("Token name:", token.name());
        console.log("Token symbol:", token.symbol());
        console.log("Initial supply:", token.totalSupply());
        console.log("Deployer balance:", token.balanceOf(deployerAddress));
    }
}
```

### Конфигурация окружения

Создайте файл `.env` с вашей конфигурацией:

```bash .env
PRIVATE_KEY=your_private_key_here
BASE_SEPOLIA_RPC_URL=https://sepolia.base.org
BASE_MAINNET_RPC_URL=https://mainnet.base.org
BASESCAN_API_KEY=your_basescan_api_key_here
```

Обновите `foundry.toml` для конфигурации сети Base:

```toml foundry.toml
[profile.default]
src = "src"
out = "out"
libs = ["lib"]
remappings = ["@openzeppelin/=lib/openzeppelin-contracts/"]

[rpc_endpoints]
base_sepolia = "${BASE_SEPOLIA_RPC_URL}"
base_mainnet = "${BASE_MAINNET_RPC_URL}"

[etherscan]
base_sepolia = { key = "${BASESCAN_API_KEY}", url = "https://api-sepolia.basescan.org/api" }
base = { key = "${BASESCAN_API_KEY}", url = "https://api.basescan.org/api" }
```

### Тестирование

Создайте комплексные тесты для вашего токена:

```solidity test/MyToken.t.sol
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.19;

import {Test, console} from "forge-std/Test.sol";
import {MyToken} from "../src/MyToken.sol";

contract MyTokenTest is Test {
    MyToken public token;
    address public owner = address(0x1);
    address public user = address(0x2);
    
    uint256 constant INITIAL_SUPPLY = 100_000_000 * 10**18;
    
    function setUp() public {
        // Развертываем контракт токена перед каждым тестом
        vm.prank(owner);
        token = new MyToken("Test Token", "TEST", INITIAL_SUPPLY, owner);
    }
    
    function testInitialState() public {
        // Проверяем, что токен развернут с правильными параметрами
        assertEq(token.name(), "Test Token");
        assertEq(token.symbol(), "TEST");
        assertEq(token.totalSupply(), INITIAL_SUPPLY);
        assertEq(token.balanceOf(owner), INITIAL_SUPPLY);
    }
    
    function testMinting() public {
        uint256 mintAmount = 1000 * 10**18;
        
        // Только владелец должен иметь возможность минтить
        vm.prank(owner);
        token.mint(user, mintAmount);
        
        assertEq(token.balanceOf(user), mintAmount);
        assertEq(token.totalSupply(), INITIAL_SUPPLY + mintAmount);
    }
    
    function testBurning() public {
        uint256 burnAmount = 1000 * 10**18;
        
        // Владелец сжигает свои токены
        vm.prank(owner);
        token.burn(burnAmount);
        
        assertEq(token.balanceOf(owner), INITIAL_SUPPLY - burnAmount);
        assertEq(token.totalSupply(), INITIAL_SUPPLY - burnAmount);
    }
    
    function testFailMintExceedsMaxSupply() public {
        // Этот тест должен падать при попытке минтить больше максимальной эмиссии
        uint256 excessiveAmount = token.MAX_SUPPLY() + 1;
        
        vm.prank(owner);
        token.mint(user, excessiveAmount);
    }
    
    function testFailUnauthorizedMinting() public {
        // Этот тест должен падать, когда не-владелец пытается минтить
        vm.prank(user);
        token.mint(user, 1000 * 10**18);
    }
}
```

Запустите ваши тесты:

```bash Terminal
# Запускаем все тесты с подробным выводом
forge test -vv
```

### Развертывание и верификация

Разверните в тестовой сети Base Sepolia:

```bash Terminal
# Загружаем переменные окружения
source .env

# Развертываем в Base Sepolia с автоматической верификацией
forge script script/DeployToken.s.sol:DeployToken \
    --rpc-url base_sepolia \
    --broadcast \
    --verify
```

<Tip>
Флаг `--verify` автоматически верифицирует ваш контракт на BaseScan, упрощая взаимодействие пользователей с вашим токеном.
</Tip>

<Note>
Чтобы развернуть в Base Mainnet, просто измените `base_sepolia` на `base_mainnet` в вашей команде развертывания. Убедитесь, что у вас достаточно ETH в основной сети Base для развертывания и оплаты комиссий.
</Note>

## Соображения после запуска

После развертывания вашего токена вот ключевые следующие шаги, которые стоит рассмотреть:

### Распределение токенов и экономика

Внимательно продумайте настройки эмиссии и распределения вашего токена. Продумайте, как токены будут распределяться между вашим сообществом, командой и участниками экосистемы. Учтите такие факторы, как графики вестинга, проценты распределения и долгосрочное согласование интересов.

### Сообщество и присутствие в соцсетях

Создайте сообщество и присутствие в соцсетях вокруг вашего токена и проекта. Это включает создание документации, настройку аккаунтов в социальных сетях, взаимодействие с экосистемой Base и построение отношений с другими проектами и разработчиками.

### Ликвидность и торговля

Добавьте ликвидность на децентрализованные биржи, такие как Uniswap, чтобы обеспечить возможность торговли. Обратите внимание, что платформы для запуска токенов обычно автоматически обрабатывают это за вас, но для пользовательских развертываний вам нужно будет создавать торговые пары и предоставлять начальную ликвидность.

### Продолжение разработки

Для комплексного руководства по развитию вашего проекта на Base, включая стратегии маркетинга, интеграцию с экосистемой и тактики роста, посетите [Base Launch Playbook](https://www.launchonbase.xyz/).

<Note>
Всегда уделяйте приоритетное внимание безопасности, прозрачности и ценности для сообщества при разработке и запуске токенов. Рассмотрите возможность проведения аудитов безопасности и следования рекомендациям для распределения токенов.
</Note>
