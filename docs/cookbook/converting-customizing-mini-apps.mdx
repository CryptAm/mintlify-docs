# Добавьте MiniKit в ваше приложение

У вас есть чистый фронтенд. Теперь превратите его в мини-приложение. Инструменты для создания кода "по настроению" (vibe coding) отлично подходят для UI, но они пока не идеально обрабатывают ончейн части. Загрузите ваш код, откройте его в Cursor или Claude Code и добавьте MiniKit, чтобы включить Smart Wallet, бесплатные транзакции (gasless) и интеграцию с Base App.

<Tip>
  Перед тем как давать промт вашему AI IDE, составьте список мест, где вы хотите,
  чтобы происходили ончейн взаимодействия. Примеры: минт (создание) коллекционного 
  предмета на экране Create, чаевые (tip) в ленте (Feed), обмен (swap) на экране Trade. 
  Это поможет AI разместить компоненты MiniKit в правильных файлах.
</Tip>

<Steps>
  <Step title='Откройте ваш проект в AI IDE'>
    Запустите Cursor или Claude Code и откройте ваш Next.js проект. Убедитесь сначала, 
    что он запускается локально с помощью <code>npm run dev</code>, чтобы AI мог 
    отталкиваться от работающей основы.
  </Step>
  <Step title='Предоставьте AI контекст из документации'>
    Используйте страницу интеграции MiniKit для существующих приложений. 
    Используйте выпадающий список на сайте для “Copy page as Markdown for LLMs”, 
    затем вставьте этот markdown в вашу AI IDE в качестве контекста.
  </Step>
  <Step title='Попросите AI проанализировать ваше приложение'>
    Пусть AI составит карту ваших маршрутов (routes), макета (layout) и структуры компонентов. 
    Попросите его предложить точные точки интеграции для провайдера и хуков MiniKit.
  </Step>
  <Step title='Интегрируйте провайдер MiniKit и Smart Wallet'>
    Добавьте провайдер MiniKit в <code>app/layout.tsx</code> (или ваш главный макет), 
    подключите базовые хуки и убедитесь, что кнопка "логин" не добавлена. Мини-приложения 
    должны ощущаться нативными и не требующими сессии (sessionless).
  </Step>
  <Step title='Добавьте Paymaster для бесплатного UX'>
    Настройте Paymaster от Coinbase Developer Platform с переменными окружения. 
    Обновите ваши потоки действий (action flows) для спонсирования транзакций, затем протестируйте локально.
  </Step>
  <Step title='Проверьте и закоммитьте'>
    Запустите ваше приложение, протестируйте каждую точку ончейн взаимодействия 
    и закоммитьте изменения с четкими сообщениями. Вы готовы к деплою.
  </Step>
</Steps>

## Полезные промты:

<Tip>
  **Короткий путь:** На любой странице документации Base используйте выпадающий список 
  для “Copy page as Markdown for LLMs” и вставьте его прямо в ваш AI чат. 
  Это дает модели точные инструкции и уменьшает количество галлюцинаций.
</Tip>
